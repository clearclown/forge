# Forge — 概念与愿景

## 问题不在于分布式推理

[mesh-llm](https://github.com/michaelneale/mesh-llm)、Petals 和 Exo 等项目已经表明，你可以通过网络在多个设备上拆分 LLM 推理。流水线并行、专家分片和网格协调等硬核工程问题已基本解决。

未解决的问题是：**为什么会有人贡献他们的硬件？**

mesh-llm 完美地汇集了 GPU——但如果你作为网格节点运行 Mac Mini 一年，你将一无所获。没有贡献记录，没有优先访问权，没有经济回报。网络靠善意运行。善意无法规模化。

## 洞察：计算即货币

每种货币体系都以稀缺性为支撑。黄金因地质原因而稀缺。石油因开采消耗能源而稀缺。比特币因挖掘在 SHA-256 哈希上燃烧电力而稀缺。

但比特币的稀缺性是人为的——计算是没有目的的。哈希保护了账本，但不产生任何有用的东西。

LLM 推理则不同。当 Forge 节点消耗电力来回答某人的问题时，该计算具有**内在价值**。有人非常渴望得到那个答案，以至于发出了请求。电力没有被浪费——它产生了智能。

```
比特币: 电力 → 无用的哈希 → 人为稀缺性 → 价值
Forge:  电力 → 有用的推理 → 真实效用 → 价值
```

这就是**计算本位制 (Compute Standard)**：一种价值单位由经过验证的有用计算支撑的货币体系。

## Forge 是什么

Forge 是带有经济体系的 mesh-llm。

推理层（网络、模型分发、API）来自 mesh-llm。Forge 添加了：

1. **CU 账本** — 每次推理都会产生一笔交易。提供者赚取 CU，消费者支出 CU。由双方双重签名。
2. **动态定价** — 每个 token 的 CU 价格随本地供需波动。空闲节点越多 → 越便宜。请求越多 → 越贵。
3. **有用工作证明** — CU 是通过执行真实的推理赚取的，而不是通过解决任意的谜题。
4. **代理预算 API** — AI 代理可以查询其余额、估算成本并做出自主支出决策。
5. **外部桥接** — CU 可以选择性地通过协议外的适配器层兑换为比特币 (Lightning)、稳定币或法定货币。

## 为什么不直接使用比特币？

我们考虑过将比特币/Lightning 作为主要的结算层。但我们决定不这么做。

| 担忧 | 解释 |
|---------|-------------|
| **哲学上的不一致** | 在以无用工作支撑的货币中奖励有用工作 |
| **外部依赖** | 如果比特币的安全性崩溃（量子计算、监管），Forge 的经济也会随之崩溃 |
| **效率** | 对于每次推理的微支付，Lightning 通道管理是额外的负担 |
| **自给自足** | CU 具有价值是因为计算本身有用——它不需要外部验证 |

对于需要外部流动性的操作员，比特币仍然可以作为**离场通道**。但协议的原生经济运行在 CU 上。

## 为什么 CU 具有价值

CU 不是投机性代币。它是对**未来计算能力的请求权**。

如果你通过提供推理赚取了 10,000 CU，你可以花费这些 CU 从网络上的任何其他节点购买推理。价值不是抽象的——它是让机器为你思考的能力。

这使得 CU 成为一种**生产性资产**，而不是价值储存手段：

```
公寓楼                      Forge 上的 Mac Mini
───────────────────         ──────────────────
资产: 建筑物                 资产: 计算硬件
成本: 维护                   成本: 电力
收入: 租金                   收入: 推理产生的 CU
收益: 租金 - 维护            收益: 赚取的 CU - 电力
空闲 = 收入损失              空闲 = 潜力浪费
```

与比特币（数字黄金——保持价值但不产生任何东西）不同，CU 就像出租物业——它通过执行有用工作来产生收益。

## 作为经济主体的 AI 代理

Forge 经济最重要的消费者不是人类——而是 AI 代理。

运行小型本地模型（手机上的 1.5B 参数）的代理智能有限。但如果它可以通过借出空闲计算赚取 CU，并花费 CU 访问更大的模型，它就可以自主地扩展自己的能力：

```
小型代理 (手机, 1.5B)
  → 夜间空闲 → 借出 CPU → 赚取 CU
  → 早上: 需要复杂的推理
  → 检查 /v1/forge/balance → 有 5,000 CU
  → 检查 /v1/forge/pricing → 70B 模型 500 个 token 消耗 2,000 CU
  → 购买 70B 推理 → 获得更聪明的回答
  → 使用回答做出更好的交易决策
  → 下一周期赚取更多 CU
```

这就是自我强化循环：做出良好经济决策的代理会变得更强，从而使其能够做出更好的决策。

无需人类批准单笔交易。代理在业主设置的预算政策内运行。协议提供市场；代理提供策略。

## 比较

| 项目 | 推理 | 经济体系 | 代理自主权 |
|---------|-----------|---------|----------------|
| **mesh-llm** | 分布式 (流水线 + MoE) | 无 | 仅黑板消息 |
| **Petals** | 分布式 (协作) | 无 | 无 |
| **Ollama** | 仅限本地 | 无 | 无 |
| **Together AI** | 中心化 | 按 token 付费 (企业) | 仅 API 访问 |
| **比特币** | 不适用 | PoW (无用工作) | 无 |
| **Golem** | 批量计算 | GNT 代币 | 人类主导 |
| **Forge** | 分布式 (mesh-llm) | **CU (有用工作)** | **自主预算管理** |

## 隐喻

一颗种子落入网络中。它通过在夜间借出空闲周期赚取第一笔 CU。有了这些 CU，它购买了访问更大型模型的权限。它变得更聪明。它找到了更高效的交易。更多的 CU。更大的模型。一片森林从一颗种子中诞生——不是因为有人播种，而是因为经济规则使增长成为必然。
