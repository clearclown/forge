# Forge — 引导启动序列

## 概述

Forge 有两条引导启动路径：

- **当前参考流程**: 由操作员驱动的显式流程。
- **目标流程**: 基于 mesh-llm 的节点加入网格并自动开始赚取 CU。

## 当前参考流程

```text
1. 启动模型主机: forge seed -m "qwen2.5:0.5b"
2. 复制打印出来的公钥
3. 连接请求者: forge worker --seed <seed_public_key>
4. 检查状态: forge status --url http://127.0.0.1:3000
5. 检查 CU 余额: curl http://127.0.0.1:3000/v1/forge/balance
```

HTTP API 默认绑定到 `127.0.0.1`。如果公开，请设置 `--api-token`。

## 目标引导启动 (mesh-llm 分叉)

一旦 Forge 与 mesh-llm 集成：

```text
1. forge --auto                          # 加入最佳公共网格
2. forge --model Qwen2.5-32B --publish   # 创建公共网格，赚取 CU
3. forge --join <token>                  # 使用 GPU 加入，赚取 CU
4. forge --client --join <token>         # 作为消费者加入，支出 CU
```

提供的每次推理都会赚取 CU。消耗的每次推理都会支出 CU。经济层是自动的——无需单独配置。

## 经济引导启动

### 新节点 (零余额)

```text
1. 节点加入网格
2. 免费层级: 立即获得 1,000 CU
3. 节点提供第一次推理请求 → 赚取 CU
4. 随着提供每次请求，CU 余额增长
5. 节点现在可以为其他节点的推理支出 CU
```

### 现有节点 (已有余额)

```text
1. 节点重启，加载持久化的账本 (forge-ledger.json)
2. 验证 HMAC-SHA256 完整性
3. 恢复之前的余额、交易和声誉
4. 节点恢复赚取和支出 CU
```

## 降级与恢复

| 事件 | 经济影响 | 推理影响 |
|---|---|---|
| 1 个远程节点断开连接 | 剩余节点吸收工作，CU 流动继续 | 短暂暂停，模型重新平衡 |
| 所有远程节点断开连接 | CU 经济暂停，仅限本地模式 | 回退到本地小模型 |
| 节点低电量 (<20%) | 停止提供服务（赚取暂停），仍可消费 | 将层级卸载到远程 |
| 节点重新获得网络 | 恢复赚取 CU，声誉恢复 | 重新发现对等节点，重新扩展 |

**关键不变量**: 节点的 CU 余额在重启和断开连接后保持不变。赚取的 CU 永远不会丢失。

## 节点贡献模型

- **贡献者**: 提供推理的设备赚取 CU
- **消费者**: 请求推理的设备支出 CU
- **余额**: 更多贡献 → 更多 CU → 更多访问他人计算资源的机会
- **免费层级**: 为新节点提供 1,000 CU，从第一次请求开始消耗
- **收益**: 在线节点每小时赚取 0.1% 的收益（按声誉加权）
- **无强制付款**: 协议运行在 CU 上。外部桥接（Lightning、法币）是可选的。

## 引导启动期间的安全性

- 在任何网络活动之前创建 Ed25519 身份
- 所有连接通过 QUIC + Noise 加密
- 在当前的播种/工人流程中，播种者可以看到提示文本（显式信任边界）
- CU 交易在本地记录，并具有 HMAC-SHA256 完整性保护
- 目标: 双重签名的交易在网格中通过 gossip 同步
