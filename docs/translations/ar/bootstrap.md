# Forge — تسلسل الإقلاع (Bootstrap Sequence)

## نظرة عامة (Overview)

يحتوي Forge على مسارين للإقلاع:

- **تدفق المرجع الحالي (current reference flow)**، وهو تدفق صريح يوجهه المشغل.
- **التدفق المستهدف (target flow)**، حيث تنضم عقدة مبنية على mesh-llm إلى الشبكة وتبدأ في كسب وحدات الحوسبة (CU) تلقائياً.

## تدفق المرجع الحالي (Current Reference Flow)

```text
1. تشغيل مستضيف النموذج: forge seed -m "qwen2.5:0.5b"
2. نسخ المفتاح العام المطبوع
3. توصيل طالب الخدمة (Requester): forge worker --seed <seed_public_key>
4. التحقق من الحالة: forge status --url http://127.0.0.1:3000
5. التحقق من رصيد CU: curl http://127.0.0.1:3000/v1/forge/balance
```

ترتبط واجهة برمجة التطبيقات (HTTP API) بـ `127.0.0.1` افتراضياً. إذا تم تعريضها للشبكة، قم بتعيين `--api-token`.

## الإقلاع المستهدف (Target Bootstrap - فرع mesh-llm)

بمجرد تكامل Forge مع mesh-llm:

```text
1. forge --auto                          # الانضمام إلى أفضل شبكة عامة
2. forge --model Qwen2.5-32B --publish   # إنشاء شبكة عامة، كسب CU
3. forge --join <token>                  # الانضمام بـ GPU، كسب CU
4. forge --client --join <token>         # الانضمام كمستهلك، إنفاق CU
```

كل استنتاج يتم تقديمه يكسب CU. كل استنتاج يتم استهلاكه ينفق CU. الطبقة الاقتصادية تلقائية — لا حاجة لتكوين منفصل.

## الإقلاع الاقتصادي (Economic Bootstrap)

### عقدة جديدة (رصيد صفر)

```text
1. العقدة تنضم للشبكة
2. الفئة المجانية: 1,000 CU متاحة فوراً
3. العقدة تقدم أول طلب استنتاج ← تكسب CU
4. رصيد CU ينمو مع كل طلب يتم تقديمه
5. العقدة يمكنها الآن إنفاق CU على استنتاج العقد الأخرى
```

### عقدة موجودة (لديها رصيد)

```text
1. العقدة تعيد التشغيل، تحمل دفتر الحسابات المحفوظ (forge-ledger.json)
2. يتم التحقق من تكامل HMAC-SHA256
3. استعادة الرصيد السابق، الصفقات، والسمعة
4. العقدة تستأنف كسب وإنفاق CU
```

## التدهور والاستعادة (Degradation & Recovery)

| الحدث | التأثير الاقتصادي | تأثير الاستنتاج |
|---|---|---|
| انفصال عقدة واحدة بعيدة | العقد المتبقية تمتص العمل، تدفق CU يستمر | توقف قصير، إعادة توازن النموذج |
| انفصال جميع العقد البعيدة | اقتصاد CU يتوقف، وضع محلي فقط | الرجوع لنموذج محلي صغير |
| بطارية العقدة منخفضة (<20%) | التوقف عن التقديم (كسب المال يتوقف)، لا يزال بالإمكان الاستهلاك | نقل الطبقات إلى عقد بعيدة |
| العقدة تستعيد الشبكة | استئناف كسب CU، استعادة السمعة | إعادة اكتشاف الأقران، إعادة التوسع |

**قاعدة ثابتة رئيسية**: رصيد CU للعقدة يستمر عبر إعادات التشغيل والانفصالات. وحدات الحوسبة المكتسبة لا تضيع أبداً.

## نموذج المساهمة في العقدة (Node Contribution Model)

- **المساهمون (Contributors)**: الأجهزة التي تقدم الاستنتاج تكسب CU.
- **المستهلكون (Consumers)**: الأجهزة التي تطلب الاستنتاج تنفق CU.
- **الرصيد**: مساهمة أكثر ← CU أكثر ← وصول أكبر لحوسبة الآخرين.
- **الفئة المجانية**: 1,000 CU للعقد الجديدة، تُستهلك من الطلب الأول.
- **العائد (Yield)**: العقد المتصلة تكسب 0.1% عائد في الساعة (مرجح بالسمعة).
- **لا دفع إلزامي**: يعمل البروتوكول على وحدات CU. الجسور الخارجية (Lightning، نقد) اختيارية.

## الأمن أثناء الإقلاع

- يتم إنشاء هوية Ed25519 قبل أي نشاط للشبكة.
- جميع الاتصالات مشفرة عبر QUIC + Noise.
- في تدفق seed/worker الحالي، يرى الـ seed نص المطالبة (prompt) (حدود ثقة صريحة).
- يتم تسجيل صفقات CU محلياً مع حماية التكامل HMAC-SHA256.
- المستهدف: صفقات موقعة مزدوجة ومزامنة Gossip عبر الشبكة.
