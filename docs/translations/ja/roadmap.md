# Forge — ロードマップ

## フェーズ 1: ローカル推論 ✅

- `forge-core`: 型定義 (NodeId, LayerRange, ModelManifest, PeerCapability)
- `forge-infer`: llama.cpp エンジン、GGUF ローダー、ストリーミングトークン生成
- `forge-node`: HTTP API (/chat, /chat/stream, /health)
- `forge-cli`: モデルの自動ダウンロード機能を備えた `forge chat` コマンド

## フェーズ 2: P2P プロトコル ✅

- `forge-net`: Iroh トランスポート、Noise 暗号化、ピア接続
- `forge-proto`: 14 種類の通信プロトコルメッセージ (bincode + 長さプレフィックス)
- `forge-node`: シード/ワーカーパイプライン、推論リクエスト/レスポンス
- 統合テスト: 2 つのノード間での Hello + 複数メッセージの交換

## フェーズ 3: リモート推論 + オペレーター台帳 ✅

- `forge-ledger`: CU 会計、取引実行、評判、利回り、市場価格設定
- `forge-node`: 推論パイプラインに統合された台帳
- 推論前の CU 残高チェック
- 完了後の取引記録
- HMAC-SHA256 による台帳の整合性保護

## フェーズ 4: 経済 API ✅

- OpenAI 互換 API: `POST /v1/chat/completions`, `GET /v1/models`
- CU メータリング: すべての推論で `x_forge` 拡張を含む取引を記録
- エージェント予算エンドポイント: `GET /v1/forge/balance`, `GET /v1/forge/pricing`
- CU→Lightning 決済ブリッジ: `forge settle --pay`
- HF Hub からのシードモデルの自動解決
- 台帳の永続化を伴う Ctrl-C での正常終了

## フェーズ 5: mesh-llm フォークの統合 (次期フェーズ)

**目標:** Forge の推論層を、実績のある mesh-llm 分散エンジンに置き換える。

| 成果物 | 説明 |
|---|---|
| mesh-llm をフォーク | 経済層が組み込まれた mesh-llm のフォークとして forge を作成 |
| forge-ledger の統合 | mesh-llm の推論パイプラインに CU 記録をフック |
| 経済 API の維持 | 新しいコードベースでも `/v1/forge/*` エンドポイントを維持 |
| Web コンソールの拡張 | mesh-llm のコンソールに CU 残高と取引の可視化を追加 |
| パイプライン + MoE | mesh-llm のパイプライン並列化と専門家シャッディングを継承 |
| Nostr 探索 | mesh-llm の公開メッシュ探索を継承 |
| CREDITS.md | mesh-llm への謝辞を文書化 |

## フェーズ 6: 有益な仕事の証明 (Proof of Useful Work)

**目標:** ネットワーク全体で CU の主張を検証可能にする。

| 成果物 | 説明 |
|---|---|
| 二重署名プロトコル | プロバイダーとコンシューマーの両方が各 TradeRecord に署名 |
| ゴシップ同期 | 署名された取引をメッシュ全体に伝播 |
| 不正検知 | 未署名または不一致の取引を拒否 |
| 評判のゴシップ | ピア間で評判スコアを共有 |
| 結託耐性 | 取引パターンの統計的異常検知 |

## フェーズ 7: 外部ブリッジ

**目標:** オペレーターが CU を外部価値に変換できるようにする。

| 成果物 | 説明 |
|---|---|
| Lightning ブリッジ | LDK を介した自動的な CU→sats 決済 |
| ステーブルコインアダプター | CU→USDC/USDT 変換 |
| 法定通貨アダプターインターフェース | 銀行振込決済の仕様策定 |
| 交換レートサービス | 公開の CU/BTC および CU/USD レートフィード |
| Bitcoin アンカリング | オプション: 定期的なマニフェストハッシュを OP_RETURN で記録し、不変の監査証跡を作成 |

## フェーズ 8: エージェント自律経済

**目標:** AI エージェントが自身の計算ライフサイクルを管理できるようにする。

| 成果物 | 説明 |
|---|---|
| 予算ポリシー | 人間が設定するエージェントごとの支出制限 |
| 自律取引 | エージェントが計算資源の売買タイミングを決定 |
| マルチモデルルーティング | コスト/品質のトレードオフに基づいたエージェントのモデル選択 |
| 自己強化 | CU を稼ぐ → より大きなモデルへのアクセス権を購入 → さらに CU を稼ぐ |
| エージェント間経済 | エージェント間での特化型計算資源の取引 (例: コードモデル vs チャットモデル) |

## 長期目標

| マイルストーン | 説明 |
|---|---|
| SDK リリース | 安定した API を備えた組み込み可能な Rust ライブラリとしての forge-node |
| プロトコル v2 | v1 の教訓を活かした、後方互換性のある進化 |
| クロスアーキテクチャ | NVIDIA GPU, AMD ROCm, RISC-V サポート (mesh-llm 経由) |
| 連合学習 (Federated Training) | 推論だけでなく、分散型ファインチューニングの実現 |
| 計算デリバティブ | 将来の計算能力に関する先渡契約 |

> プロトコルがプラットフォームであり、計算が通貨である。
