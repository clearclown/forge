# Forge — コンセプトとビジョン

## 課題は「分散推論」ではない

[mesh-llm](https://github.com/michaelneale/mesh-llm)、Petals、Exo といったプロジェクトは、ネットワークを介して複数のデバイスで LLM 推論を分割できることを証明しました。パイプライン並列化、専門家シャッディング（Expert Sharding）、メッシュ調整といった高度なエンジニアリングは、大部分が解決されています。

未解決の課題は、**「なぜ誰かが自分のハードウェアを提供するのか？」**ということです。

mesh-llm は GPU を美しくプールしますが、Mac Mini をメッシュノードとして1年間稼働させても、何も得られません。貢献の記録も、優先アクセス権も、経済的なリターンもありません。ネットワークは善意で動いています。善意はスケールしません。

## 洞察: 計算は通貨である

すべての通貨システムは希少性に裏打ちされています。金は地質学的に希少です。石油は抽出にエネルギーコストがかかるため希少です。ビットコインは SHA-256 ハッシュの計算に電力を消費するため希少です。

しかし、ビットコインの希少性は人工的であり、その計算には目的がありません。ハッシュは台帳を保護しますが、有用なものは何も生み出しません。

LLM 推論は異なります。Forge ノードが誰かの質問に答えるために電力を消費するとき、その計算には**固有の価値**があります。誰かがその回答を必要としてリクエストしたからです。電力は浪費されず、知能を生み出しました。

```
Bitcoin:  電力 → 無意味なハッシュ → 人工的な希少性 → 価値
Forge:    電力 → 有用な推論       → 本物のユーティリティ → 価値
```

これが**計算本位制 (Compute Standard)** です。価値の単位が、検証された有用な計算によって裏打ちされる通貨システムです。

## Forge とは何か

Forge は「経済層を備えた mesh-llm」です。

推論層（ネットワーキング、モデル配布、API）は mesh-llm から提供されます。Forge は以下を追加します：

1. **CU 台帳 (CU Ledger)** — すべての推論が取引を生み出します。プロバイダーは CU を稼ぎ、消費者は CU を支払います。両当事者による二重署名が行われます。
2. **動的価格設定 (Dynamic Pricing)** — トークンあたりの CU 価格は、現地の需要と供給に応じて変動します。アイドル状態のノードが多いほど安くなり、リクエストが多いほど高くなります。
3. **有益な仕事の証明 (Proof of Useful Work)** — CU は任意のパズルを解くのではなく、実際の推論を実行することで獲得されます。
4. **エージェント予算 API (Agent Budget API)** — AI エージェントは自分の残高を確認し、コストを見積もり、自律的な支出決定を行うことができます。
5. **外部ブリッジ (External Bridges)** — オプションとして、プロトコル外のアダプター層を介して CU をビットコイン（Lightning）、ステーブルコイン、または法定通貨と交換できます。

## なぜビットコインをそのまま使わないのか？

ビットコイン/Lightning を主要な決済層にすることを検討しましたが、採用しないことに決めました。

| 懸念点 | 説明 |
|---------|-------------|
| **哲学的な不一致** | 無意味な仕事に裏打ちされた通貨で、有益な仕事に報酬を与えることの矛盾 |
| **外部依存** | ビットコインのセキュリティが損なわれた場合（量子計算、規制など）、Forge の経済も崩壊する |
| **効率性** | 推論ごとのマイクロペイメントにおいて、Lightning チャネルの管理はオーバーヘッドになる |
| **自給自足** | CU は計算自体が有用であるために価値を持つ。外部の検証を必要としない |

ビットコインは、外部の流動性を必要とするオペレーターのための**オフランプ（出口）**として引き続き利用可能です。しかし、プロトコルのネイティブ経済は CU で動作します。

## なぜ CU に価値があるのか

CU は投機的なトークンではありません。それは**将来の計算に対する請求権**です。

推論を提供して 10,000 CU を獲得した場合、その CU を使ってネットワーク上の他のノードから推論を購入できます。価値は抽象的なものではなく、マシンに自分のために考えさせる能力そのものです。

これにより、CU は価値の保存手段ではなく、**生産的資産**となります：

```
賃貸アパート                    Forge 上の Mac Mini
───────────────────         ──────────────────
資産: 建物                    資産: 計算ハードウェア
コスト: 維持費                コスト: 電気代
収益: 家賃                    収益: 推論による CU
利回り: 家賃 - 維持費         利回り: 獲得 CU - 電気代
空室 = 収入の喪失             アイドル = 潜在能力の浪費
```

ビットコイン（デジタルゴールド — 価値は維持するが何も生み出さない）とは異なり、CU は賃貸物件のようなものです。有用な仕事を遂行することで利回りを生み出します。

## 経済主体としての AI エージェント

Forge 経済の最も重要な消費者は人間ではなく、AI エージェントです。

小規模なローカルモデル（スマホ上の 1.5B パラメータなど）を実行しているエージェントは、知能が限られています。しかし、アイドル状態の計算資源を貸し出して CU を稼ぎ、その CU を使ってより大きなモデルにアクセスできれば、自律的に自身の能力を拡張できます：

```
小規模エージェント (スマホ, 1.5B)
  → 夜間にアイドル状態 → CPU を貸し出す → CU を稼ぐ
  → 朝: 複雑な推論が必要になる
  → /v1/forge/balance を確認 → 5,000 CU 保有
  → /v1/forge/pricing を確認 → 70B モデルが 500 トークンで 2,000 CU
  → 70B の推論を購入 → より賢い回答を得る
  → その回答を使ってより良い取引判断を行う
  → 次のサイクルでさらに CU を稼ぐ
```

これが自己強化ループです。経済的に優れた意思決定を行うエージェントはより強く成長し、それによってさらに優れた意思決定が可能になります。

人間が個々の取引を承認する必要はありません。エージェントは所有者によって設定された予算ポリシーの範囲内で動作します。プロトコルは市場を提供し、エージェントは戦略を提供します。

## 比較

| プロジェクト | 推論 | 経済層 | エージェントの自律性 |
|---------|-----------|---------|----------------|
| **mesh-llm** | 分散型 (パイプライン + MoE) | なし | ブラックボード通信のみ |
| **Petals** | 分散型 (協調的) | なし | なし |
| **Ollama** | ローカルのみ | なし | なし |
| **Together AI** | 中央集権型 | トークン課金 (企業) | API アクセスのみ |
| **Bitcoin** | N/A | PoW (無意味な仕事) | なし |
| **Golem** | バッチ計算 | GNT トークン | 人間の指示が必要 |
| **Forge** | 分散型 (mesh-llm) | **CU (有益な仕事)** | **自律的な予算管理** |

## メタファー

一粒の種がネットワークに落ちます。それは夜間にアイドルサイクルを貸し出すことで、最初の CU を稼ぎます。その CU で、より大きなモデルへのアクセスを購入します。種はより賢くなります。より効率的な取引を見つけます。さらに CU が増えます。さらに大きなモデル。誰かが植えたからではなく、経済原理が成長を不可避にしたために、一粒の種から森が生まれるのです。
