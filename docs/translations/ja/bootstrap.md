# Forge — ブートストラップ手順

## 概要

Forge には 2 つのブートストラップパスがあります：

- **現在のリファレンスフロー**: オペレーター主導の明示的な手順。
- **ターゲットフロー**: mesh-llm ベースのノードがメッシュに参加し、自動的に CU を稼ぎ始める手順。

## 現在のリファレンスフロー

```text
1. モデルホストを起動: forge seed -m "qwen2.5:0.5b"
2. 表示された公開キーをコピー
3. リクエスターを接続: forge worker --seed <seed_public_key>
4. ステータスを確認: forge status --url http://127.0.0.1:3000
5. CU 残高を確認: curl http://127.0.0.1:3000/v1/forge/balance
```

HTTP API はデフォルトで `127.0.0.1` にバインドされます。外部に公開する場合は `--api-token` を設定してください。

## ターゲットブートストラップ (mesh-llm フォーク)

Forge が mesh-llm と完全に統合された場合：

```text
1. forge --auto                          # 最良の公開メッシュに参加
2. forge --model Qwen2.5-32B --publish   # 公開メッシュを作成し、CU を稼ぐ
3. forge --join <token>                  # GPU で参加し、CU を稼ぐ
4. forge --client --join <token>         # コンシューマーとして参加し、CU を使う
```

提供されたすべての推論は CU を獲得します。消費されたすべての推論は CU を支払います。経済層は自動化されており、個別の設定は不要です。

## 経済的ブートストラップ

### 新規ノード (残高ゼロ)

```text
1. ノードがメッシュに参加
2. フリーティア: 1,000 CU が即座に利用可能
3. ノードが最初の推論リクエストを提供 → CU を獲得
4. 提供するリクエストごとに CU 残高が増加
5. ノードは他のノードの推論に CU を支出可能になる
```

### 既存ノード (残高あり)

```text
1. ノードが再起動し、保存された台帳 (forge-ledger.json) をロード
2. HMAC-SHA256 による整合性を検証
3. 以前の残高、取引、および評判が復元される
4. ノードは CU の獲得と支出を再開
```

## 劣化と回復

| イベント | 経済的影響 | 推論への影響 |
|---|---|---|
| 1 つのリモートノードが切断 | 残りのノードが作業を吸収、CU フローは継続 | 一時的な停止、モデルの再構成 |
| すべてのリモートノードが切断 | CU 経済が一時停止、ローカル専用モード | ローカルの小規模モデルにフォールバック |
| ノードのバッテリー低下 (<20%) | 提供を停止 (獲得が停止)、消費は可能 | レイヤーをリモートにオフロード |
| ノードがネットワークに復帰 | CU の獲得を再開、評判が回復 | ピアを再探索、再拡張 |

**重要な不変条件**: ノードの CU 残高は再起動や切断をまたいで維持されます。獲得した CU が失われることはありません。

## ノードの貢献モデル

- **貢献者 (Contributors)**: 推論を提供するデバイスは CU を獲得。
- **消費者 (Consumers)**: 推論をリクエストするデバイスは CU を支出。
- **残高 (Balance)**: 貢献が増えるほど CU が増え、他者の計算資源へのアクセス権が増える。
- **フリーティア (Free tier)**: 新規ノードに 1,000 CU を付与。最初のリクエストから消費される。
- **利回り (Yield)**: オンラインのノードは 1 時間あたり 0.1% の利回りを獲得（評判により加重）。
- **強制的な支払いはなし**: プロトコルは CU で動作。外部ブリッジ（Lightning、法定通貨）はオプション。

## ブートストラップ時のセキュリティ

- ネットワーク活動の前に Ed25519 アイデンティティを作成。
- すべての接続は QUIC + Noise で暗号化。
- 現在のシード/ワーカーフローでは、シードはプロンプトのテキストを閲覧可能（明示的な信頼境界）。
- CU 取引は HMAC-SHA256 の整合性を伴ってローカルに記録される。
- ターゲット: 二重署名された取引がメッシュ全体でゴシップ同期される。
