# Forge — アーキテクチャ

## 概要

Forge は、**推論 (Inference)** と **経済 (Economy)** の2つの層からなるシステムです。

推論層は、モデルの配布、メッシュネットワーク、および API サービングを処理します。これは [mesh-llm](https://github.com/michaelneale/mesh-llm) に基づいて構築されています。

経済層は、CU 会計、取引記録、価格設定、およびエージェント予算を処理します。これが Forge 独自の貢献です。

```
┌─────────────────────────────────────────────────┐
│  SDK / 統合境界                                 │
│  任意のクライアントが forge-node をライブラリとして組み込み可能 │
│  サードパーティエージェント、ダッシュボード、アダプター       │
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│  経済層 (Forge 独自)                            │
│                                                  │
│  ┌──────────────┐ ┌──────────┐ ┌─────────────┐ │
│  │ forge-ledger │ │ pricing  │ │ agent       │ │
│  │ CU 取引      │ │ 供給/需要│ │ 予算管理    │ │
│  │ 評判         │ │          │ │ /v1/forge/* │ │
│  │ 利回り       │ │          │ │             │ │
│  └──────────────┘ └──────────┘ └─────────────┘ │
│                                                  │
│  ┌──────────────┐ ┌──────────────────────────┐  │

│  └──────────────┘ └──────────────────────────┘  │

                   │
┌──────────────────▼──────────────────────────────┐

│                                                  │
│  ┌────────────┐ ┌───────────┐ ┌──────────────┐ │
│  │ iroh メッシュ│ │ llama.cpp │ │ OpenAI API   │ │
│  │ QUIC+Noise │ │ パイプライン│ │ /v1/chat/    │ │
│  │ Nostr 探索  │ │ MoE 分散   │ │ completions  │ │
```

## 推論層 (mesh-llm)

推論層の役割：

- **メッシュネットワーク**: iroh ベースの Noise 暗号化を伴う QUIC 接続。
- **ピア探索**: 公開メッシュ用の Nostr リレー、LAN 用の mDNS。
- **モデル配布**: 密なモデル用のパイプライン並列化、MoE 用の専門家シャッディング。
- **推論実行**: llama-server および rpc-server サブプロセスを介した llama.cpp。
- **API サービング**: OpenAI 互換の `/v1/chat/completions` および `/v1/models`。

Forge はこれらすべてを mesh-llm から継承しています。推論層は、CU、取引、または価格設定については関知しません。

## 経済層 (Forge)

経済層は推論層の上に位置し、以下の役割を担います：

### forge-ledger — 経済エンジン

```rust
pub struct ComputeLedger {
    balances: HashMap<NodeId, NodeBalance>,
    work_log: Vec<WorkUnit>,
    trade_log: Vec<TradeRecord>,
    price: MarketPrice,
}
```

主な役割：
- ノードごとの CU 残高（貢献量、消費量、予約量）を追跡。
- すべての推論取引（プロバイダー、コンシューマー、CU 量、トークン数）を記録。
- 需要と供給から動的な市場価格を算出。
- 貢献ノードに利回りを適用。
- プロトコル外ブリッジ用の決済明細をエクスポート。
- HMAC-SHA256 による整合性保護を伴うスナップショットの保存。

### forge-verify — 有益な仕事の証明 (開発目標)

CU の主張が正当であることを保証します：
- 二重署名プロトコル: プロバイダーとコンシューマーの両方が各 TradeRecord に署名。
- ゴシップ同期: 署名された取引がネットワーク全体に伝播。
- 検証: どのノードでも両方の署名を検証可能。
- 不正検知: 不一致または未署名の取引を拒否。

### forge-bridge — 外部決済 (オプション)

オペレーターのために CU を外部価値に変換します：
- Bitcoin Lightning: 設定された交換レートで CU → msats。
- ステーブルコイン: アダプターを介した CU → USDC/USDT。
- 法定通貨: オペレーターダッシュボードを介した CU → 銀行振込。

ブリッジ層はコアプロトコルの外部にあります。オペレーターごとに異なるブリッジを使用できます。

### API サーフェス

| ルート | レイヤー | 説明 |
|-------|-------|-------------|
| `POST /v1/chat/completions` | 推論 + 経済 | 推論を実行し、CU 取引を記録 |
| `GET /v1/models` | 推論 | ロードされたモデルの一覧 |
| `GET /v1/forge/balance` | 経済 | CU 残高、評判 |
| `GET /v1/forge/pricing` | 経済 | 市場価格、コスト見積もり |
| `GET /status` | 経済 | 市場価格、ネットワーク統計、最近の取引 |
| `GET /topology` | 推論 | モデルマニフェスト、ピア、シャード計画 |
| `GET /settlement` | 経済 | エクスポート可能な取引履歴 |
| `GET /health` | 推論 | 基本的なヘルスチェック |

## データフロー

### CU 会計を伴う推論

```
コンシューマーがリクエストを送信
    ↓
API が POST /v1/chat/completions を受信
    ↓
台帳が確認: can_afford(consumer, estimated_cost)?
    ↓ はい
推論層が実行 (llama-server / rpc-server)
    ↓
トークンがコンシューマーにストリーム返送される
    ↓
台帳が取引を記録:
  - provider.contributed += cu_cost
  - consumer.consumed += cu_cost
  - trade_log.push(TradeRecord)
    ↓
レスポンスに x_forge を付与: { cu_cost, effective_balance }
```

### 決済データのエクスポート

```
オペレーターが実行: forge settle --hours 24
    ↓
API が指定期間の trade_log を読み取り
    ↓
ノードごとに集計: 総獲得量、総消費量、純 CU
    ↓
参照価格を含む JSON 明細をエクスポート
    ↓
オペレーターがブリッジアダプターを使用して純 CU を BTC/法定通貨に変換
```

## セキュリティモデル

```
レイヤー 0: Bitcoin メインチェーン   ← オプションのアンカリング (将来)
レイヤー 1: 二重署名               ← 各取引にプロバイダーとコンシューマーが署名
レイヤー 2: HMAC-SHA256 台帳      ← ローカルの整合性保護
レイヤー 3: iroh (QUIC + Noise)    ← 転送路の暗号化
レイヤー 4: 推論実行               ← プロバイダー上でモデルをローカル実行
```

各層が異なる脅威から保護します：
- レイヤー 4: モデルの整合性 (GGUF ハッシュ検証)
- レイヤー 3: 転送の秘匿性 (盗聴)
- レイヤー 2: ローカルな改ざん (ファイル修正)
- レイヤー 1: ネットワーク不正 (虚偽の CU 主張)
- レイヤー 0: 歴史的な不変性 (オプションの Bitcoin アンカー)

## クレートの依存関係

```
forge-core ← 共有型定義 (NodeId, CU, Config)
    ↑
forge-ledger ← 経済エンジン (取引、価格設定、利回り)
    ↑
forge-lightning ← 外部ブリッジ (LDK ウォレット、CU↔sats)
    ↑
forge-node ← オーケストレーター (HTTP API、パイプライン、台帳統合)
    ↑
forge-cli ← リファレンス CLI (チャット、シード、ワーカー、決済)

forge-net ← P2P 転送 (iroh, QUIC, Noise, mDNS)
forge-proto ← 通信メッセージ (bincode, 14 種類のペイロード)
forge-infer ← 推論エンジン (llama.cpp, GGUF ローダー)
forge-shard ← トポロジー計画 (レイヤー割り当て、再構成)
```
