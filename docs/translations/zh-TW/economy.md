# Forge — 經濟模型

## 計算本位制 (Compute Standard)

每種貨幣體系都以稀缺性為支撐：

| 時代 | 標準 | 支撐 |
|-----|----------|---------|
| 古代 | 金/銀 | 地質稀缺性 |
| 1944–1971 | 布雷頓森林體系 | 美元掛鉤黃金 |
| 1971–現在 | 石油美元 | 石油需求 + 軍事力量 |
| 2009–現在 | 比特幣 | 燃燒在 SHA-256 上的電力 |
| **Forge** | **計算本位制** | **花費在有用推理上的電力** |

Forge 引入了計算本位制：價值單位由執行有用計算的真實能源支出支撐。與比特幣的工作量證明不同，Forge 中花費的每一焦耳都會產生真實的智能。

## CU: 原生貨幣

### 什麼是 CU

**1 CU = 驗證過的 10 億次浮點運算 (FLOPs) 的推理工作量。**

CU 不是加密貨幣。它不是區塊鏈上的代幣。它是一個代表已執行的真實計算量的核算單位。CU 具有價值，是因為它是對未來計算能力的請求權——如果你透過提供推理賺取了 CU，你可以花費它來換取推理服務。

### 為什麼選擇 CU，而不是比特幣

| 屬性 | CU | 比特幣 |
|----------|-----|---------|
| **價值支撐** | 有用計算 (內在價值) | 雜湊計算 (人為價值) |
| **結算速度** | 即時 (本地帳本) | 秒到分鐘 (Lightning/鏈上) |
| **交易成本** | 零 | 通道費、鏈上費 |
| **外部依賴** | 無 | 比特幣網絡健康狀況 |
| **量子風險** | 無 (無加密謎題) | SHA-256 / ECDSA 易受攻擊 |
| **收益產生** | 是 (空閑硬體賺取 CU) | 否 (錢包中的 BTC 不產生收益) |

CU 是**主要**結算單位。比特幣、穩定幣和法幣是透過協定外的橋接適配器提供的可選**離場通道**。

### 作為生產性資產的 CU

```
公寓樓                      Forge 上的 Mac Mini
───────────────────         ──────────────────
資產: 建築物                 資產: 計算硬體
成本: 維護                   成本: 電力
收入: 租金                   收入: 推理產生的 CU
收益: 租金 - 維護            收益: 賺取的 CU - 電力成本
空閑 = 收入損失              空閑 = 潛力浪費
```

Forge 上的計算設備不同於錢包裡的比特幣（靜態價值，無收益）。它就像出租物業——透過執行有用工作產生收入。

## 交易模型

### 交易執行

每次推理都會在兩方之間產生一筆交易：

```rust
pub struct TradeRecord {
    pub provider: NodeId,       // 誰運行了推理
    pub consumer: NodeId,       // 誰請求了推理
    pub cu_amount: u64,         // 轉移的 CU 金額
    pub tokens_processed: u64,  // 完成的工作量
    pub timestamp: u64,
    pub model_id: String,
}
```

交易由雙方記錄。在當前實現中，每個節點維護一個本地帳本。目標實現將增加雙重簽名和 gossip 同步。

### 動態定價

CU 價格根據本地供需關係浮動：

```
有效價格 = 基礎 CU/token × 需求因子 / 供應因子
```

- **空閑節點越多** → 供應因子上升 → 價格下跌
- **推理請求越多** → 需求因子上升 → 價格上漲
- 每個節點觀察自己的市場狀況。沒有全球訂單簿。

### 免費層級

沒有貢獻歷史的新節點將獲得 1,000 CU。這讓任何人都能立即使用網絡。免費層級從第一次請求開始消耗，且不會重置。

女巫攻擊 (Sybil) 緩解：如果帳本中存在超過 100 個從未貢獻過的未知節點（貢獻 = 0, 消耗 > 0），新的免費層級請求將被拒絕。

## 有用工作證明 (Proof of Useful Work)

### 概念

比特幣的工作量證明：「我燃燒電力計算了 SHA-256 雜湊。這是證明它的隨機數 (nonce)。」

Forge 的有用工作證明：「我燃燒電力運行了 LLM 推理。這是回應，這是消費者確認收到回應的簽名。」

關鍵區別：比特幣的證明是自生成的（任何礦工都可以產生有效的雜湊）。Forge 的證明需要一個**交易對手**——真正需要該推理的人。你無法偽造需求。

### 驗證協定 (目標)

```
1. 消費者向提供者發送 InferenceRequest
2. 提供者執行推理，將 token 流回
3. 消費者收到 token，計算回應雜湊
4. 雙方為 TradeRecord 簽名：
   - 提供者簽名：「我計算了這些」
   - 消費者簽名：「我收到了這些」
5. 雙重簽名的 TradeRecord 透過 gossip 同步到網絡
6. 任何節點都可以驗證雙方簽名
```

節點無法在沒有協作對手的情況下增加其 CU 餘額。共謀是可能的，但在經濟上是不合理的——共謀的消費者透過簽署虛假交易得不到任何好處。

### 當前實現

當前的參考實現使用具有 HMAC-SHA256 完整性保護的本地帳本。雙重簽名和 gossip 是下一步。

## 收益與聲譽

### 收益

保持在線並貢獻計算資源的節點將賺取收益：

```
收益 CU = 貢獻 CU × 0.001 × 聲譽 × 在線時長 (小時)
```

在聲譽為 1.0 時，一個貢獻了 10,000 CU 的節點在 8 小時的夜間可以賺取 80 CU。這不是通貨膨脹——這是對可用性的獎勵。可靠在線的節點對網絡更有價值。

### 聲譽

每個節點都有一個介於 0.0 到 1.0 之間的聲譽分數：

- 新節點從 0.5 開始
- 在線時間和成功的交易會增加聲譽
- 斷開連接和失敗的交易會降低聲譽
- 更高的聲譽 → 更高的收益率，調度中的優先級更高

## 結算與外部橋接

### 核心規則

**協定以 CU 結算。** 轉換為其他任何東西都是集成層面的考量。

### 結算報表

操作員可以導出任何時間窗口的可審計交易歷史：

```
forge settle --hours 24 --price 0.05 --out settlement.json
```

報表包括：總收益 CU、總支出 CU、淨 CU、交易計數，以及可選的每 CU 參考價格。

### 橋接架構

```
第 0 層: Forge 協定
  → CU 核算、交易、定價

第 1 層: 結算報表
  → 可導出的交易歷史
  → 參考匯率

第 2 層: 外部橋接 (可選)
  → CU ↔ BTC (Lightning)
  → CU ↔ 穩定幣
  → CU ↔ 法幣
```

橋接層位於協定之外。不同的操作員可以使用不同的橋接。即使外部流動性為零，協定仍然有用。

### Lightning 橋接

對於希望對比特幣結算的操作員：

```bash
forge settle --hours 24 --pay
```

這將根據淨收益 CU，按配置的匯率（預設：每 CU 10 msats）創建一個 BOLT11 Lightning 發票。

## 代理主導的預算管理

### 願景

傳統模式：人類決定 → 人類支付 → AI 執行
Forge 模式：政策允許代理 → 代理檢查預算 → 代理支出 CU → 代理執行

### API

```
GET /v1/forge/balance   → CU 餘額、貢獻、消耗、聲譽
GET /v1/forge/pricing   → 市場價格，每 100/1000 token 的成本估算
```

代理可以：
1. 在發出請求前檢查其餘額
2. 根據當前市場價格估算推理成本
3. 決定請求是否值得花費 CU
4. 自動執行並支付

人類主管設置預算政策。代理在這些限制內自主運行。

### 自我強化循環

```
代理 (小型, 手機端)
  → 透過借出空閑計算賺取 CU
  → 花費 CU 購買大型模型訪問權限
  → 變得更聰明
  → 做出更好的經濟決策
  → 賺取更多 CU
  → 訪問更大型的模型
  → ...
```

這是一種可能的應用模式。協定提供市場；代理提供策略。

## 為什麼這不是 Web3

大多數 Web3 專案在豐富的數位商品之上創造人為稀缺性（代幣）。Forge 則相反：

- **計算資源確實稀缺** — 它需要真實的電力、真實的矽片、真實的時間
- **CU 不是投機性的** — 它代表經過驗證的工作，而不是對未來的賭注
- **沒有 ICO，沒有代幣銷售，沒有治理代幣** — CU 是透過工作賺取的
- **無需區塊鏈** — 雙邊簽名和 gossip 即可
- **沒有智能合約** — 協定本身就是合同

價值是由物理規律製造的，而不是由共識製造的。
