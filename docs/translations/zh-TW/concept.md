# Forge — 概念與願景

## 問題不在於分散式推理

[mesh-llm](https://github.com/michaelneale/mesh-llm)、Petals 和 Exo 等專案已經表明，你可以透過網絡在多個設備上拆分 LLM 推理。流水線並行、專家分片和網格協調等硬核工程問題已基本解決。

未解決的問題是：**為什麼會有人貢獻他們的硬體？**

mesh-llm 完美地匯集了 GPU——但如果你作為網格節點運行 Mac Mini 一年，你將一無所獲。沒有貢獻記錄，沒有優先訪問權，沒有經濟回報。網絡靠善意運行。善意無法規模化。

## 洞察：計算即貨幣

每種貨幣體系都以稀缺性為支撐。黃金因地質原因而稀缺。石油因開採消耗能源而稀缺。比特幣因挖掘在 SHA-256 雜湊上燃燒電力而稀缺。

但比特幣的稀缺性是人為的——計算是沒有目的的。雜湊保護了帳本，但不產生任何有用的東西。

LLM 推理則不同。當 Forge 節點消耗電力來回答某人的問題時，該計算具有**內在價值**。有人非常渴望得到那個答案，以至於發出了請求。電力沒有被浪費——它產生了智能。

```
比特幣: 電力 → 無用的雜湊 → 人為稀缺性 → 價值
Forge:  電力 → 有用的推理 → 真實效用 → 價值
```

這就是**計算本位制 (Compute Standard)**：一種價值單位由經過驗證的有用計算支撐的貨幣體系。

## Forge 是什麼

Forge 是帶有經濟體系的 mesh-llm。

推理層（網絡、模型分發、API）來自 mesh-llm。Forge 添加了：

1. **CU 帳本** — 每次推理都會產生一筆交易。提供者賺取 CU，消費者支出 CU。由雙方雙重簽名。
2. **動態定價** — 每個 token 的 CU 價格隨本地供需波動。空閒節點越多 → 越便宜。請求越多 → 越貴。
3. **有用工作證明** — CU 是透過執行真實的推理賺取的，而不是透過解決任意的謎題。
4. **代理預算 API** — AI 代理可以查詢其餘額、估算成本並做出自主支出決策。
5. **外部橋接** — CU 可以選擇性地透過協定外的適配器層兌換為比特幣 (Lightning)、穩定幣或法定貨幣。

## 為什麼不直接使用比特幣？

我們考慮過將比特幣/Lightning 作為主要的結算層。但我們決定不這麼做。

| 擔憂 | 解釋 |
|---------|-------------|
| **哲學上的不一致** | 在以無用工作支撐的貨幣中獎勵有用工作 |
| **外部依賴** | 如果比特幣的安全性崩潰（量子計算、監管），Forge 的經濟也會隨之崩潰 |
| **效率** | 對於每次推理的微支付，Lightning 通道管理是額外的負擔 |
| **自給自足** | CU 具有價值是因為計算本身有用——它不需要外部驗證 |

對於需要外部流動性的操作員，比特幣仍然可以作為**離場通道**。但協定的原生經濟運行在 CU 上。

## 為什麼 CU 具有價值

CU 不是投機性代幣。它是對**未來計算能力的請求權**。

如果你透過提供推理賺取了 10,000 CU，你可以花費這些 CU 從網絡上的任何其他節點購買推理。價值不是抽象的——它是讓機器為你思考的能力。

這使得 CU 成為一種**生產性資產**，而不是價值儲存手段：

```
公寓樓                      Forge 上的 Mac Mini
───────────────────         ──────────────────
資產: 建築物                 資產: 計算硬體
成本: 維護                   成本: 電力
收入: 租金                   收入: 推理產生的 CU
收益: 租金 - 維護            收益: 賺取的 CU - 電力
空閑 = 收入損失              空閑 = 潛力浪費
```

與比特幣（數位黃金——保持價值但不產生任何東西）不同，CU 就像出租物業——它透過執行有用工作來產生收益。

## 作為經濟主體的 AI 代理

Forge 經濟最重要的消費者不是人類——而是 AI 代理。

運行小型本地模型（手機上的 1.5B 參數）的代理智能有限。但如果它可以透過借出空閑計算賺取 CU，並花費 CU 訪問更大的模型，它就可以自主地擴展自己的能力：

```
小型代理 (手機, 1.5B)
  → 夜間空閑 → 借出 CPU → 賺取 CU
  → 早上: 需要複雜的推理
  → 檢查 /v1/forge/balance → 有 5,000 CU
  → 檢查 /v1/forge/pricing → 70B 模型 500 個 token 消耗 2,000 CU
  → 購買 70B 推理 → 獲得更聰明的回答
  → 使用回答做出更好的交易決策
  → 下一週期賺取更多 CU
```

這就是自我強化循環：做出良好經濟決策的代理會變得更強，從而使其能夠做出更好的決策。

無需人類批准單筆交易。代理在業主設置的預算政策內運行。協定提供市場；代理提供策略。

## 比較

| 專案 | 推理 | 經濟體系 | 代理自主權 |
|---------|-----------|---------|----------------|
| **mesh-llm** | 分散式 (流水線 + MoE) | 無 | 僅黑板消息 |
| **Petals** | 分散式 (協作) | 無 | 無 |
| **Ollama** | 僅限本地 | 無 | 無 |
| **Together AI** | 中心化 | 按 token 付費 (企業) | API 訪問而已 |
| **比特幣** | 不適用 | PoW (無用工作) | 無 |
| **Golem** | 批量計算 | GNT 代幣 | 人類主導 |
| **Forge** | 分散式 (mesh-llm) | **CU (有用工作)** | **自主預算管理** |

## 隱喻

一顆種子落入網絡中。它透過在夜間借出空閑週期賺取第一筆 CU。有了這些 CU，它購買了訪問更大型模型的權限。它變得更聰明。它找到了更高效的交易。更多的 CU。更大的模型。一片森林從一顆種子中誕生——不是因為有人播種，而是因為經濟規則使增長成為必然。
