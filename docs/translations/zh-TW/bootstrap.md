# Forge — 引導啟動序列

## 概述

Forge 有兩條引導啟動路徑：

- **當前參考流程**: 由操作員驅動的顯式流程。
- **目標流程**: 基於 mesh-llm 的節點加入網格並自動開始賺取 CU。

## 當前參考流程

```text
1. 啟動模型主機: forge seed -m "qwen2.5:0.5b"
2. 複製打印出來的公鑰
3. 連接請求者: forge worker --seed <seed_public_key>
4. 檢查狀態: forge status --url http://127.0.0.1:3000
5. 檢查 CU 餘額: curl http://127.0.0.1:3000/v1/forge/balance
```

HTTP API 預設綁定到 `127.0.0.1`。如果公開，請設置 `--api-token`。

## 目標引導啟動 (mesh-llm 分叉)

一旦 Forge 與 mesh-llm 集成：

```text
1. forge --auto                          # 加入最佳公共網格
2. forge --model Qwen2.5-32B --publish   # 創建公共網格，賺取 CU
3. forge --join <token>                  # 使用 GPU 加入，賺取 CU
4. forge --client --join <token>         # 作為消費者加入，支出 CU
```

提供的每次推理都會賺取 CU。消耗的每次推理都會支出 CU。經濟層是自動的——無需單獨配置。

## 經濟引導啟動

### 新節點 (零餘額)

```text
1. 節點加入網格
2. 免費層級: 立即獲得 1,000 CU
3. 節點提供第一次推理請求 → 賺取 CU
4. 隨著提供每次請求，CU 餘額增長
5. 節點現在可以為其他節點的推理支出 CU
```

### 現有節點 (已有餘額)

```text
1. 節點重啟，加載持久化的帳本 (forge-ledger.json)
2. 驗證 HMAC-SHA256 完整性
3. 恢復之前的餘額、交易和聲譽
4. 節點恢復賺取和支出 CU
```

## 降級與恢復

| 事件 | 經濟影響 | 推理影響 |
|---|---|---|
| 1 個遠端節點斷開連接 | 剩餘節點吸收工作，CU 流動繼續 | 短暫暫停，模型重新平衡 |
| 所有遠端節點斷開連接 | CU 經濟暫停，僅限本地模式 | 回退到本地小模型 |
| 節點低電量 (<20%) | 停止提供服務（賺取暫停），仍可消費 | 將層級卸載到遠端 |
| 節點重新獲得網絡 | 恢復賺取 CU，聲譽恢復 | 重新發現對等節點，重新擴展 |

**關鍵不變量**: 節點的 CU 餘額在重啟和斷開連接後保持不變。賺取的 CU 永遠不會丟失。

## 節點貢獻模型

- **貢獻者**: 提供推理的設備賺取 CU
- **消費者**: 請求推理的設備支出 CU
- **餘額**: 更多貢獻 → 更多 CU → 更多訪問他人計算資源的機會
- **免費層級**: 為新節點提供 1,000 CU，從第一次請求開始消耗
- **收益**: 在線節點每小時賺取 0.1% 的收益（按聲譽加權）
- **無強制付款**: 協定運行在 CU 上。外部橋接（Lightning、法幣）是可選的。

## 引導啟動期間的安全性

- 在任何網絡活動之前創建 Ed25519 身份
- 所有連接透過 QUIC + Noise 加密
- 在當前的播種/工人流程中，播種者可以看到提示文本（顯式信任邊界）
- CU 交易在本地記錄，並具有 HMAC-SHA256 完整性保護
- 目標: 雙重簽名的交易在網格中透過 gossip 同步
