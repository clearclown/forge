# Forge — आर्किटेक्चर (Architecture)

## अवलोकन (Overview)

Forge एक दो-परत वाली प्रणाली (two-layer system) है: **इन्फरेंस (inference)** और **अर्थव्यवस्था (economy)**।

इन्फरेंस परत मॉडल वितरण, मेश नेटवर्किंग और API सेवा का प्रबंधन करती है। यह [mesh-llm](https://github.com/michaelneale/mesh-llm) पर आधारित है।

आर्थिक परत CU अकाउंटिंग, ट्रेड रिकॉर्डिंग, मूल्य निर्धारण और एजेंट बजट का प्रबंधन करती है। यह Forge का मौलिक योगदान है।

```
┌─────────────────────────────────────────────────┐
│  SDK / एकीकरण सीमा (Integration Boundary)        │
│  कोई भी क्लाइंट forge-node को लाइब्रेरी के रूप में  │
│  शामिल कर सकता है। तृतीय-पक्ष एजेंट, डैशबोर्ड, एडॉप्टर।│
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│  आर्थिक परत (Economic Layer - Forge-original)  │
│                                                  │
│  ┌──────────────┐ ┌──────────┐ ┌─────────────┐ │
│  │ forge-ledger │ │ मूल्य     │ │ एजेंट       │ │
│  │ CU ट्रेड      │ │ निर्धारण   │ │ बजट        │ │
│  │ प्रतिष्ठा      │ │ आपूर्ति/   │ │ /v1/forge/* │ │
│  │ प्रतिफल       │ │ मांग       │ │             │ │
│  └──────────────┘ └──────────┘ └─────────────┘ │
│                                                  │
│  ┌──────────────┐ ┌──────────────────────────┐  │
│  │ forge-verify │ │ forge-bridge (वैकल्पिक)    │  │
│  │ द्विपक्षीय हस्ताक्षर│ │ CU ↔ BTC Lightning      │  │
│  │ गॉसिप सिंक     │ │ CU ↔ stablecoin         │  │
│  └──────────────┘ └──────────────────────────┘  │
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│  इन्फरेंस परत (Inference Layer - mesh-llm-derived)│
│                                                  │
│  ┌────────────┐ ┌───────────┐ ┌──────────────┐ │
│  │ iroh मेश    │ │ llama.cpp │ │ OpenAI API   │ │
│  │ QUIC+Noise │ │ पाइपलाइन    │ │ /v1/chat/    │ │
│  │ Nostr डिस्कवरी│ │ MoE शार्ड   │ │ completions  │ │
└────────────┘ └───────────┘ └──────────────┘ │
└─────────────────────────────────────────────────┘
```

## इन्फरेंस परत (Inference Layer - mesh-llm)

इन्फरेंस परत निम्नलिखित के लिए जिम्मेदार है:

- **मेश नेटवर्किंग (Mesh networking)**: Noise एन्क्रिप्शन के साथ iroh-आधारित QUIC कनेक्शन।
- **पीयर डिस्कवरी (Peer discovery)**: सार्वजनिक मेश के लिए Nostr रिले, LAN के लिए mDNS।
- **मॉडल वितरण (Model distribution)**: डेंस (dense) मॉडल के लिए पाइपलाइन समानता, MoE के लिए एक्सपर्ट शार्डिंग।
- **इन्फरेंस निष्पादन (Inference execution)**: llama-server और rpc-server सब-प्रोसेस के माध्यम से llama.cpp।
- **API सेवा (API serving)**: OpenAI-संगत `/v1/chat/completions` और `/v1/models` एंडपॉइंट।

Forge यह सब mesh-llm से विरासत में प्राप्त करता है। इन्फरेंस परत को CU, ट्रेड या मूल्य निर्धारण के बारे में कोई जानकारी नहीं होती है।

## आर्थिक परत (Economic Layer - Forge)

आर्थिक परत इन्फरेंस के ऊपर स्थित होती है और निम्नलिखित के लिए जिम्मेदार है:

### forge-ledger — आर्थिक इंजन (The Economic Engine)

```rust
pub struct ComputeLedger {
    balances: HashMap<NodeId, NodeBalance>,
    work_log: Vec<WorkUnit>,
    trade_log: Vec<TradeRecord>,
    price: MarketPrice,
}
```

मुख्य जिम्मेदारियाँ:
- प्रति-नोड CU बैलेंस (योगदान, उपभोग, आरक्षित) को ट्रैक करना।
- हर इन्फरेंस ट्रेड (प्रदाता, उपभोक्ता, CU मात्रा, टोकन) को रिकॉर्ड करना।
- आपूर्ति/मांग से गतिशील बाजार मूल्यों की गणना करना।
- योगदान देने वाले नोड्स पर प्रतिफल (yield) लागू करना।
- ऑफ-प्रोटोकॉल ब्रिज के लिए सेटलमेंट स्टेटमेंट निर्यात करना।
- HMAC-SHA256 अखंडता के साथ स्नैपशॉट को डिस्क पर सहेजना।

### forge-verify — उपयोगी कार्य का प्रमाण (Proof of Useful Work)

यह सुनिश्चित करता है कि CU दावे वैध हैं:
- द्विपक्षीय हस्ताक्षर प्रोटोकॉल: प्रदाता और उपभोक्ता दोनों प्रत्येक TradeRecord पर हस्ताक्षर करते हैं।
- गॉसिप सिंक: हस्ताक्षरित ट्रेड पूरे नेटवर्क में प्रसारित होते हैं।
- सत्यापन: कोई भी नोड दोनों हस्ताक्षरों को सत्यापित कर सकता है।
- धोखाधड़ी का पता लगाना: बेमेल या बिना हस्ताक्षरित ट्रेडों को खारिज कर दिया जाता है।

### forge-bridge — बाहरी सेटलमेंट (External Settlement - वैकल्पिक)

उन ऑपरेटरों के लिए CU को बाहरी मूल्य में परिवर्तित करता है जिन्हें इसकी आवश्यकता है:
- बिटकॉइन लाइटनिंग: विन्यास योग्य विनिमय दर के माध्यम से CU → msats।
- स्टेबलकॉइन: एडॉप्टर के माध्यम से CU → USDC/USDT।
- फिएट (Fiat): ऑपरेटर डैशबोर्ड के माध्यम से CU → बैंक ट्रांसफर।

ब्रिज परत कोर प्रोटोकॉल के बाहर है। अलग-अलग ऑपरेटर अलग-अलग ब्रिज का उपयोग कर सकते हैं।

### API सतह (API Surface)

| रूट (Route) | परत | विवरण |
|-------|-------|-------------|
| `POST /v1/chat/completions` | इन्फरेंस + आर्थिक | इन्फरेंस चलाएं, CU ट्रेड रिकॉर्ड करें |
| `GET /v1/models` | इन्फरेंस | लोड किए गए मॉडल की सूची |
| `GET /v1/forge/balance` | आर्थिक | CU बैलेंस, प्रतिष्ठा |
| `GET /v1/forge/pricing` | आर्थिक | बाजार मूल्य, लागत अनुमान |
| `GET /status` | आर्थिक | बाजार मूल्य, नेटवर्क आँकड़े, हालिया ट्रेड |
| `GET /topology` | इन्फरेंस | मॉडल मेनिफेस्ट, पीयर्स, शार्ड प्लान |
| `GET /settlement` | आर्थिक | निर्यात योग्य ट्रेड इतिहास |
| `GET /health` | इन्फरेंस | बुनियादी स्वास्थ्य जांच |

## डेटा प्रवाह (Data Flow)

### CU अकाउंटिंग के साथ इन्फरेंस

```
उपभोक्ता अनुरोध भेजता है
    ↓
API को POST /v1/chat/completions प्राप्त होता है
    ↓
लेज़र जांचता है: क्या उपभोक्ता खर्च वहन कर सकता है (can_afford)?
    ↓ हाँ
इन्फरेंस परत निष्पादित करती है (llama-server / rpc-server)
    ↓
टोकन उपभोक्ता के पास स्ट्रीम होते हैं
    ↓
लेज़र ट्रेड रिकॉर्ड करता है:
  - प्रदाता का योगदान += cu_cost
  - उपभोक्ता का उपभोग += cu_cost
  - trade_log.push(TradeRecord)
    ↓
रिस्पांस में x_forge शामिल होता है: { cu_cost, effective_balance }
```

### सेटलमेंट निर्यात (Settlement Export)

```
ऑपरेटर चलाता है: forge settle --hours 24
    ↓
API समय सीमा के लिए trade_log पढ़ता है
    ↓
प्रति-नोड गणना: कुल अर्जित, कुल खर्च, शुद्ध CU
    ↓
वैकल्पिक संदर्भ मूल्य के साथ JSON स्टेटमेंट निर्यात करता है
    ↓
ऑपरेटर शुद्ध CU को BTC/fiat में बदलने के लिए ब्रिज एडॉप्टर का उपयोग करता है
```

## सुरक्षा मॉडल (Security Model)

```
परत 0: बिटकॉइन मेनचेन      ← वैकल्पिक एंकरिंग (भविष्य में)
परत 1: द्विपक्षीय हस्ताक्षर     ← प्रदाता + उपभोक्ता प्रत्येक ट्रेड पर हस्ताक्षर करते हैं
परत 2: HMAC-SHA256 लेज़र  ← स्थानीय अखंडता सुरक्षा
परत 3: iroh (QUIC + Noise) ← ट्रांसपोर्ट एन्क्रिप्शन
परत 4: इन्फरेंस निष्पादन       ← प्रदाता पर स्थानीय रूप से मॉडल चलता है
```

प्रत्येक परत विभिन्न खतरों से रक्षा करती है:
- परत 4: मॉडल अखंडता (GGUF हैश सत्यापन)
- परत 3: ट्रांसपोर्ट गोपनीयता (ईव्सड्रॉपिंग)
- परत 2: स्थानीय छेड़छाड़ (फ़ाइल संशोधन)
- परत 1: नेटवर्क धोखाधड़ी (फर्जी CU दावे)
- परत 0: ऐतिहासिक अपरिवर्तनीयता (वैकल्पिक बिटकॉइन एंकर)

## क्रेट निर्भरताएं (Crate Dependencies)

```
forge-core ← साझा प्रकार (NodeId, CU, Config)
    ↑
forge-ledger ← आर्थिक इंजन (ट्रेड, मूल्य निर्धारण, प्रतिफल)
    ↑
forge-lightning ← बाहरी ब्रिज (LDK वॉलेट, CU↔sats)
    ↑
forge-node ← ऑर्केस्ट्रेटर (HTTP API, पाइपलाइन, लेज़र एकीकरण)
    ↑
forge-cli ← संदर्भ CLI (चैट, सीड, वर्कर, सेटलमेंट)

forge-net ← P2P ट्रांसपोर्ट (iroh, QUIC, Noise, mDNS)
forge-proto ← वायर संदेश (bincode, 14 पेलोड प्रकार)
forge-infer ← इन्फरेंस इंजन (llama.cpp, GGUF लोडर)
forge-shard ← टोपोलॉजी प्लानर (परत असाइनमेंट, रीबैलेंसिंग)
```
