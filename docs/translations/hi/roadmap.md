# Forge — रोडमैप (Roadmap)

## चरण 1: स्थानीय इन्फरेंस (Local Inference) ✅

- `forge-core`: टाइप सिस्टम (NodeId, LayerRange, ModelManifest, PeerCapability)
- `forge-infer`: llama.cpp इंजन, GGUF लोडर, स्ट्रीमिंग टोकन जनरेशन
- `forge-node`: HTTP API (/chat, /chat/stream, /health)
- `forge-cli`: मॉडल ऑटो-डाउनलोड के साथ `forge chat` कमांड

## चरण 2: P2P प्रोटोकॉल (P2P Protocol) ✅

- `forge-net`: Iroh ट्रांसपोर्ट, Noise एन्क्रिप्शन, पीयर कनेक्शन
- `forge-proto`: 14 वायर प्रोटोकॉल संदेश प्रकार (bincode + length-prefix)
- `forge-node`: सीड/वर्कर पाइपलाइन, इन्फरेंस अनुरोध/प्रतिक्रिया
- एकीकरण परीक्षण (Integration tests): 2 नोड्स के बीच Hello + कई संदेशों का आदान-प्रदान

## चरण 3: रिमोट इन्फरेंस + ऑपरेटर लेज़र (Operator Ledger) ✅

- `forge-ledger`: CU अकाउंटिंग, ट्रेड निष्पादन, प्रतिष्ठा, प्रतिफल, बाजार मूल्य निर्धारण
- `forge-node`: इन्फरेंस पाइपलाइन में लेज़र का एकीकरण
- इन्फरेंस से पहले CU बैलेंस की जांच
- काम पूरा होने के बाद ट्रेड रिकॉर्ड
- HMAC-SHA256 लेज़र अखंडता

## चरण 4: आर्थिक API (Economic API) ✅

- OpenAI-संगत API: `POST /v1/chat/completions`, `GET /v1/models`
- CU मीटरिंग: प्रत्येक इन्फरेंस `x_forge` विस्तार के साथ एक ट्रेड रिकॉर्ड करता है
- एजेंट बजट एंडपॉइंट: `GET /v1/forge/balance`, `GET /v1/forge/pricing`
- CU→Lightning सेटलमेंट ब्रिज: `forge settle --pay`
- HF Hub से सीड मॉडल ऑटो-रिज़ॉल्व
- लेज़र निरंतरता (persistence) के साथ सुरक्षित Ctrl-C शटडाउन

## चरण 5: mesh-llm फोर्क एकीकरण (अगला)

**लक्ष्य:** Forge की इन्फरेंस परत को mesh-llm के सिद्ध वितरित इंजन (distributed engine) से बदलना।

| डिलिवरेबल (Deliverable) | विवरण |
|---|---|
| mesh-llm का फोर्क | आर्थिक परत के साथ mesh-llm फोर्क के रूप में Forge बनाएं |
| forge-ledger एकीकरण | mesh-llm की इन्फरेंस पाइपलाइन में CU रिकॉर्डिंग जोड़ें |
| आर्थिक API सुरक्षित रखें | नए कोडबेस में /v1/forge/* एंडपॉइंट्स बनाए रखें |
| वेब कंसोल विस्तार | mesh-llm के कंसोल में CU बैलेंस और ट्रेड विज़िबिलिटी जोड़ें |
| पाइपलाइन + MoE | mesh-llm की पाइपलाइन समानता और एक्सपर्ट शार्डिंग अपनाएं |
| Nostr डिस्कवरी | mesh-llm की सार्वजनिक मेश डिस्कवरी अपनाएं |
| CREDITS.md | mesh-llm के श्रेय (attribution) का दस्तावेजीकरण करें |

## चरण 6: उपयोगी कार्य का प्रमाण (Proof of Useful Work)

**लक्ष्य:** पूरे नेटवर्क में CU दावों को सत्यापन योग्य बनाना।

| डिलिवरेबल (Deliverable) | विवरण |
|---|---|
| द्विपक्षीय हस्ताक्षर प्रोटोकॉल | प्रदाता और उपभोक्ता दोनों प्रत्येक TradeRecord पर हस्ताक्षर करें |
| गॉसिप सिंक | हस्ताक्षरित ट्रेड पूरे मेश में प्रसारित हों |
| धोखाधड़ी का पता लगाना | बिना हस्ताक्षरित या बेमेल ट्रेडों को अस्वीकार करें |
| प्रतिष्ठा गॉसिप | पीयर्स के बीच प्रतिष्ठा स्कोर साझा करें |
| मिलीभगत (Collusion) प्रतिरोध | ट्रेड पैटर्न पर सांख्यिकीय विसंगति का पता लगाना |

## चरण 7: बाहरी ब्रिज (External Bridges)

**लक्ष्य:** ऑपरेटरों को CU को बाहरी मूल्य में बदलने की सुविधा देना।

| डिलिवरेबल (Deliverable) | विवरण |
|---|---|
| लाइटनिंग ब्रिज | LDK के माध्यम से स्वचालित CU→sats सेटलमेंट |
| स्टेबलकॉइन एडॉप्टर | CU→USDC/USDT परिवर्तन |
| फिएट एडॉप्टर इंटरफ़ेस | बैंक-ट्रांसफर सेटलमेंट के लिए स्पेसिफिकेशन |
| विनिमय दर सेवा | सार्वजनिक CU/BTC और CU/USD दर फीड |
| बिटकॉइन एंकरिंग | वैकल्पिक: अपरिवर्तनीय ऑडिट ट्रेल के लिए समय-समय पर मर्कल रूट → OP_RETURN |

## चरण 8: एजेंट स्वायत्त अर्थव्यवस्था (Agent Autonomous Economy)

**लक्ष्य:** AI एजेंटों को अपना कंप्यूट जीवनचक्र स्वयं प्रबंधित करने देना।

| डिलिवरेबल (Deliverable) | विवरण |
|---|---|
| बजट नीतियां | प्रति-एजेंट मानव द्वारा निर्धारित खर्च सीमा |
| स्वायत्त व्यापार | एजेंट तय करे कि कंप्यूट कब खरीदना/बेचना है |
| मल्टी-मॉडल रूटिंग | लागत/गुणवत्ता संतुलन के आधार पर एजेंट मॉडल चुने |
| स्व-सुदृढीकरण | एजेंट CU कमाए → बड़े मॉडल एक्सेस खरीदे → और अधिक CU कमाए |
| अंतर-एजेंट अर्थव्यवस्था | एजेंट विशेषज्ञ कंप्यूट का व्यापार करें (कोड मॉडल बनाम चैट मॉडल) |
