# Forge — बूटस्ट्रैप अनुक्रम (Bootstrap Sequence)

## अवलोकन (Overview)

Forge के दो बूटस्ट्रैप पथ (bootstrap paths) हैं:

- **वर्तमान संदर्भ प्रवाह (current reference flow)**, जो स्पष्ट और ऑपरेटर द्वारा संचालित है।
- **लक्ष्य प्रवाह (target flow)**, जहाँ एक mesh-llm-आधारित नोड मेश (mesh) में शामिल होता है और स्वचालित रूप से CU अर्जित करना शुरू कर देता है।

## वर्तमान संदर्भ प्रवाह (Current Reference Flow)

```text
1. एक मॉडल होस्ट शुरू करें: forge seed -m "qwen2.5:0.5b"
2. प्रिंट की गई पब्लिक की (public key) को कॉपी करें
3. एक अनुरोधकर्ता (requester) को कनेक्ट करें: forge worker --seed <seed_public_key>
4. स्थिति की जांच करें: forge status --url http://127.0.0.1:3000
5. CU बैलेंस चेक करें: curl http://127.0.0.1:3000/v1/forge/balance
```

HTTP API डिफ़ॉल्ट रूप से `127.0.0.1` पर बाइंड होता है। यदि इसे सार्वजनिक किया जाता है, तो `--api-token` सेट करें।

## लक्ष्य बूटस्ट्रैप (Target Bootstrap - mesh-llm fork)

एक बार Forge, mesh-llm के साथ एकीकृत हो जाने पर:

```text
1. forge --auto                          # सर्वोत्तम सार्वजनिक मेश में शामिल हों
2. forge --model Qwen2.5-32B --publish   # सार्वजनिक मेश बनाएं, CU कमाएं
3. forge --join <token>                  # GPU के साथ जुड़ें, CU कमाएं
4. forge --client --join <token>         # उपभोक्ता के रूप में जुड़ें, CU खर्च करें
```

प्रदान की गई प्रत्येक इन्फरेंस सेवा CU अर्जित करती है। उपभोग किया गया प्रत्येक इन्फरेंस CU खर्च करता है। आर्थिक परत स्वचालित है — किसी अलग कॉन्फ़िगरेशन की आवश्यकता नहीं है।

## आर्थिक बूटस्ट्रैप (Economic Bootstrap)

### नया नोड (शून्य बैलेंस)

```text
1. नोड मेश में शामिल होता है
2. फ्री टियर: 1,000 CU तुरंत उपलब्ध
3. नोड पहला इन्फरेंस अनुरोध पूरा करता है → CU कमाता है
4. प्रत्येक पूरे किए गए अनुरोध के साथ CU बैलेंस बढ़ता है
5. नोड अब अन्य नोड्स के इन्फरेंस पर CU खर्च कर सकता है
```

### मौजूदा नोड (बैलेंस के साथ)

```text
1. नोड पुनरारंभ (restart) होता है, सहेजे गए लेज़र (forge-ledger.json) को लोड करता है
2. HMAC-SHA256 अखंडता सत्यापित की जाती है
3. पिछला बैलेंस, ट्रेड और प्रतिष्ठा बहाल की जाती है
4. नोड फिर से CU कमाना और खर्च करना शुरू करता है
```

## गिरावट और रिकवरी (Degradation & Recovery)

| घटना | आर्थिक प्रभाव | इन्फरेंस प्रभाव |
|---|---|---|
| 1 रिमोट नोड डिस्कनेक्ट होता है | शेष नोड्स कार्यभार संभालते हैं, CU प्रवाह जारी रहता है | संक्षिप्त विराम, मॉडल रीबैलेंस |
| सभी रिमोट नोड्स डिस्कनेक्ट होते हैं | CU अर्थव्यवस्था रुक जाती है, केवल स्थानीय मोड | स्थानीय छोटे मॉडल का उपयोग |
| नोड की बैटरी कम है (<20%) | सेवा देना बंद (कमाना रुकता है), उपभोग जारी रह सकता है | परतों (layers) को रिमोट पर ऑफलोड करना |
| नोड नेटवर्क वापस पाता है | फिर से CU कमाना शुरू, प्रतिष्ठा बहाल | पीयर्स की फिर से खोज, विस्तार |

**मुख्य अपरिवर्तनीय नियम (Key invariant)**: नोड का CU बैलेंस रिस्टार्ट और डिस्कनेक्शन के बाद भी बना रहता है। अर्जित CU कभी नहीं खोता है।

## नोड योगदान मॉडल (Node Contribution Model)

- **योगदानकर्ता (Contributors)**: इन्फरेंस सेवा देने वाले उपकरण CU कमाते हैं।
- **उपभोक्ता (Consumers)**: इन्फरेंस का अनुरोध करने वाले उपकरण CU खर्च करते हैं।
- **बैलेंस (Balance)**: अधिक योगदान → अधिक CU → दूसरों के कंप्यूट तक अधिक पहुँच।
- **फ्री टियर**: नए नोड्स के लिए 1,000 CU, पहले अनुरोध से ही उपभोग शुरू।
- **प्रतिफल (Yield)**: ऑनलाइन नोड्स प्रति घंटे 0.1% प्रतिफल कमाते हैं (प्रतिष्ठा के आधार पर)।
- **कोई अनिवार्य भुगतान नहीं**: प्रोटोकॉल CU पर चलता है। बाहरी ब्रिज (Lightning, fiat) वैकल्पिक हैं।

## बूटस्ट्रैप के दौरान सुरक्षा (Security During Bootstrap)

- किसी भी नेटवर्क गतिविधि से पहले Ed25519 पहचान (identity) बनाई जाती है।
- सभी कनेक्शन QUIC + Noise के माध्यम से एन्क्रिप्ट किए जाते हैं।
- वर्तमान सीड/वर्कर प्रवाह में, सीड प्रॉम्ट टेक्स्ट देख सकता है (स्पष्ट विश्वास सीमा)।
- CU ट्रेड स्थानीय रूप से HMAC-SHA256 अखंडता के साथ रिकॉर्ड किए जाते हैं।
- लक्ष्य: द्विपक्षीय हस्ताक्षरित ट्रेड पूरे मेश में गॉसिप-सिंक किए जाएं।
