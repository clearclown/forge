# Forge — Послідовність бутстрапу

## Огляд

Forge має два шляхи бутстрапу (запуску):

- **поточний референсний потік**, який є явним і керується оператором
- **цільовий потік**, де вузол на базі mesh-llm приєднується до мережі та автоматично починає заробляти CU

## Поточний референсний потік

```text
1. Запустіть хост моделі: forge seed -m "qwen2.5:0.5b"
2. Скопіюйте надрукований публічний ключ
3. Підключіть реквестера: forge worker --seed <seed_public_key>
4. Перевірте статус: forge status --url http://127.0.0.1:3000
5. Перевірте баланс CU: curl http://127.0.0.1:3000/v1/forge/balance
```

HTTP API за замовчуванням прив'язується до `127.0.0.1`. Якщо воно відкрите назовні, встановіть `--api-token`.

## Цільовий бутстрап (форк mesh-llm)

Після інтеграції Forge з mesh-llm:

```text
1. forge --auto                          # приєднатися до найкращої публічної мережі
2. forge --model Qwen2.5-32B --publish   # створити публічну мережу, заробляти CU
3. forge --join <token>                  # приєднатися з GPU, заробляти CU
4. forge --client --join <token>         # приєднатися як споживач, витрачати CU
```

Кожен наданий інференс заробляє CU. Кожен спожитий інференс витрачає CU. Економічний шар працює автоматично — окрема конфігурація не потрібна.

## Економічний бутстрап

### Новий вузол (нульовий баланс)

```text
1. Вузол приєднується до мережі
2. Безкоштовний рівень: 1000 CU доступні негайно
3. Вузол обслуговує перший запит інференсу → заробляє CU
4. Баланс CU зростає з кожним обслуженим запитом
5. Тепер вузол може витрачати CU на інференс інших вузлів
```

### Існуючий вузол (має баланс)

```text
1. Вузол перезапускається, завантажує збережений леджер (forge-ledger.json)
2. Цілісність HMAC-SHA256 перевірена
3. Попередній баланс, операції та репутація відновлені
4. Вузол продовжує заробляти та витрачати CU
```

## Деградація та відновлення

| Подія | Економічний вплив | Вплив на інференс |
|---|---|---|
| 1 віддалений вузол відключився | Інші вузли беруть роботу на себе, потік CU триває | Коротка пауза, модель ребалансована |
| Усі віддалені вузли відключилися | Економіка CU стає на паузу, режим "тільки локально" | Перехід на локальну маленьку модель |
| Низький заряд батареї (<20%) | Припинення обслуговування (заробіток на паузі), можна споживати | Перенесення шарів на віддалені вузли |
| Вузол повернувся в мережу | Продовження заробітку CU, репутація відновлюється | Повторний пошук пірів, розширення |

**Ключовий інваріант**: Баланс CU вузла зберігається після перезапусків та відключень. Зароблені CU ніколи не втрачаються.

## Модель внеску вузлів

- **Контриб'ютори**: Пристрої, що обслуговують інференс, заробляють CU
- **Споживачі**: Пристрої, що запитують інференс, витрачають CU
- **Баланс**: Більше внеску → більше CU → більше доступу до чужих обчислень
- **Безкоштовний рівень**: 1000 CU для нових вузлів, витрачаються з першого запиту
- **Дохідність**: Вузли онлайн заробляють 0.1% доходу на годину (зважено за репутацією)
- **Ніяких обов'язкових платежів**: Протокол працює на CU. Зовнішні мости (Lightning, фіат) опціональні.

## Безпека під час бутстрапу

- Ідентичність Ed25519 створюється перед будь-якою мережевою активністю
- Усі з'єднання зашифровані через QUIC + Noise
- У поточному потоці seed/worker сід бачить текст промпту (явна межа довіри)
- Операції CU записуються локально з захистом цілісності HMAC-SHA256
- Ціль: операції з подвійним підписом синхронізуються через gossip по всій мережі
