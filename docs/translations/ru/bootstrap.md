# Forge — Последовательность запуска (Bootstrap)

## Обзор

Forge имеет два пути запуска:

- **текущий эталонный поток**, который является явным и управляется оператором
- **целевой поток**, где узел на базе mesh-llm присоединяется к сети и начинает автоматически зарабатывать CU

## Текущий эталонный поток

```text
1. Запустите хост модели: forge seed -m "qwen2.5:0.5b"
2. Скопируйте напечатанный публичный ключ
3. Подключите запрашивающую сторону: forge worker --seed <seed_public_key>
4. Проверьте статус: forge status --url http://127.0.0.1:3000
5. Проверьте баланс CU: curl http://127.0.0.1:3000/v1/forge/balance
```

HTTP API по умолчанию привязывается к `127.0.0.1`. Если он открыт вовне, установите `--api-token`.

## Целевой запуск (форк mesh-llm)

После интеграции Forge с mesh-llm:

```text
1. forge --auto                          # присоединиться к лучшей публичной сети
2. forge --model Qwen2.5-32B --publish   # создать публичную сеть, зарабатывать CU
3. forge --join <token>                  # присоединиться с GPU, зарабатывать CU
4. forge --client --join <token>         # присоединиться как потребитель, тратить CU
```

Каждый обслуженный инференс приносит CU. Каждый потребленный инференс стоит CU. Экономический слой работает автоматически — отдельная настройка не требуется.

## Экономический запуск

### Новый узел (нулевой баланс)

```text
1. Узел присоединяется к сети
2. Бесплатный уровень: 1000 CU доступны немедленно
3. Узел обслуживает первый запрос инференса → зарабатывает CU
4. Баланс CU растет с каждым обслуженным запросом
5. Теперь узел может тратить CU на инференс на других узлах
```

### Существующий узел (с балансом)

```text
1. Узел перезагружается, загружает сохраненный леджер (forge-ledger.json)
2. Целостность HMAC-SHA256 проверена
3. Предыдущий баланс, сделки и репутация восстановлены
4. Узел продолжает зарабатывать и тратить CU
```

## Деградация и восстановление

| Событие | Экономическое влияние | Влияние на инференс |
|---|---|---|
| 1 удаленный узел отключился | Остальные узлы забирают работу, поток CU продолжается | Короткая пауза, ребалансировка модели |
| Все удаленные узлы отключились | Экономика CU на паузе, режим только локально | Переход на локальную малую модель |
| Низкий заряд батареи (<20%) | Остановка обслуживания (заработок на паузе), можно потреблять | Перенос слоев на удаленные узлы |
| Сеть восстановлена | Возобновление заработка CU, репутация восстанавливается | Поиск пиров, расширение |

**Ключевой инвариант**: Баланс CU узла сохраняется после перезагрузок и отключений. Заработанные CU никогда не теряются.

## Модель вклада узла

- **Участники (Contributors)**: Устройства, обслуживающие инференс, зарабатывают CU
- **Потребители (Consumers)**: Устройства, запрашивающие инференс, тратят CU
- **Баланс**: Больше вклада → больше CU → больше доступа к вычислениям других
- **Бесплатный уровень**: 1000 CU для новых узлов, расходуются с первого запроса
- **Доход (Yield)**: Узлы онлайн зарабатывают 0.1% дохода в час (взвешено по репутации)
- **Нет обязательных платежей**: Протокол работает на CU. Внешние мосты (Lightning, фиат) опциональны.

## Безопасность при запуске

- Идентификатор Ed25519 создается до любой сетевой активности
- Все соединения зашифрованы через QUIC + Noise
- В текущем потоке seed/worker сид видит текст промпта (явная граница доверия)
- Сделки CU записываются локально с проверкой целостности HMAC-SHA256
- Цель: сделки с двойной подписью, синхронизируемые через gossip по всей сети
