# Forge — Концепция и видение

## Проблема не в распределенном инференсе

Проекты типа [mesh-llm](https://github.com/michaelneale/mesh-llm), Petals и Exo показали, что инференс LLM можно распределить между несколькими устройствами по сети. Сложные инженерные задачи параллелизма пайплайна, шардинга экспертов и сетевой координации в значительной степени решены.

Нерешенная проблема такова: **почему кто-то должен предоставлять свое оборудование?**

mesh-llm отлично объединяет GPU — но если вы запускаете свой Mac Mini как узел сети в течение года, вы ничего не получаете. Ни записи о вкладе, ни приоритетного доступа, ни экономического эффекта. Сеть держится на доброй воле. Добрая воля не масштабируется.

## Инсайт: Вычисления — это деньги

Каждая денежная система подкреплена редкостью. Золото редко из-за геологии. Нефть редка, так как ее добыча стоит энергии. Bitcoin редок, потому что майнинг сжигает электричество на хеши SHA-256.

Но редкость Bitcoin искусственна — вычисления бесцельны. Хеши защищают леджер, но не создают ничего полезного.

Инференс LLM иной. Когда узел Forge тратит электричество, чтобы ответить на чей-то вопрос, это вычисление имеет **внутреннюю ценность**. Кто-то хотел получить этот ответ достаточно сильно, чтобы сделать запрос. Электричество не было потрачено впустую — оно создало интеллект.

```
Bitcoin:   электричество → бесполезное хеширование → искусств. редкость → ценность
Forge:     электричество → полезный инференс → реальная полезность → ценность
```

Это **Вычислительный стандарт (Compute Standard)**: денежная система, где единица стоимости подкреплена верифицированными полезными вычислениями.

## Что такое Forge

Forge — это mesh-llm с экономикой.

Слой инференса (сеть, распределение моделей, API) взят из mesh-llm. Forge добавляет:

1. **Леджер CU** — Каждый инференс создает сделку. Провайдер зарабатывает CU, потребитель тратит CU. Подписывается обеими сторонами.
2. **Динамическое ценообразование** — Количество CU за токен колеблется в зависимости от местного спроса и предложения. Больше свободных узлов → дешевле. Больше запросов → дороже.
3. **Доказательство полезной работы (Proof of Useful Work)** — CU зарабатываются выполнением реального инференса, а не решением произвольных головоломок.
4. **API бюджета агента** — ШИ-агенты могут запрашивать свой баланс, оценивать затраты и принимать автономные решения о расходах.
5. **Внешние мосты** — CU могут быть опционально обменены на Bitcoin (Lightning), стейблкоины или фиат через слои адаптеров вне протокола.

## Почему не просто использовать Bitcoin?

Мы рассматривали возможность сделать Bitcoin/Lightning основным расчетным слоем. Мы решили отказаться от этого.

| Опасение | Объяснение |
|---------|-------------|
| **Философское несоответствие** | Вознаграждение за полезную работу в валюте, подкрепленной бесполезной работой |
| **Внешняя зависимость** | Если безопасность Bitcoin сломается (квантовые вычисления, регуляция), экономика Forge тоже сломается |
| **Эффективность** | Управление каналами Lightning — это лишние расходы для микроплатежей за каждый инференс |
| **Самодостаточность** | CU имеет ценность, потому что сами вычисления полезны — они не требуют внешнего подтверждения |

Bitcoin остается доступным как **выход (off-ramp)** для операторов, которым нужна внешняя ликвидность. Но нативная экономика протокола работает на CU.

## Почему CU имеет ценность

CU — это не спекулятивный токен. Это **право на будущие вычисления**.

Если вы заработали 10 000 CU, обслуживая инференс, вы можете потратить эти CU на покупку инференса у любого другого узла в сети. Стоимость не абстрактна — это способность заставить машину думать за вас.

Это делает CU **продуктивным активом**, а не средством сбережения:

```
Многоквартирный дом          Mac Mini на Forge
───────────────────         ──────────────────
Актив: здание               Актив: оборудование
Затраты: обслуживание       Затраты: электричество
Доход: аренда               Доход: CU от инференса
Прибыль: аренда - обслуж.   Прибыль: CU - электричество
Простой = потеря дохода     Простой = упущенный потенциал
```

В отличие от Bitcoin (цифровое золото — сохраняет стоимость, но ничего не производит), CU подобен арендной недвижимости — он генерирует доход, выполняя полезную работу.

## ШИ-агенты как экономические субъекты

Самый важный потребитель экономики Forge — не люди, а ШИ-агенты.

Агент, работающий на малой локальной модели (1.5B параметров на телефоне), имеет ограниченный интеллект. Но если он может зарабатывать CU, одалживая свободные мощности, и тратить CU для доступа к более крупным моделям, он может автономно расширять свои возможности:

```
Малый агент (телефон, 1.5B)
  → простой ночью → одалживает CPU → зарабатывает CU
  → утро: нужно сложное рассуждение
  → проверяет /v1/forge/balance → есть 5 000 CU
  → проверяет /v1/forge/pricing → 70B модель стоит 2 000 CU за 500 токенов
  → покупает инференс 70B → получает более умный ответ
  → использует ответ для принятия лучших торговых решений
  → зарабатывает больше CU в следующем цикле
```

Это цикл самосиления: агенты, принимающие хорошие экономические решения, становятся сильнее, что позволяет им принимать еще лучшие решения.

Человеку не нужно одобрять отдельные транзакции. Агент работает в рамках бюджетной политики, установленной владельцем. Протокол предоставляет рынок; агент предоставляет стратегию.

## Сравнение

| Проект | Инференс | Экономика | Автономия агентов |
|---------|-----------|---------|----------------|
| **mesh-llm** | Распредел. (пайплайн + MoE) | Нет | Только Blackboard |
| **Petals** | Распредел. (коллаборат.) | Нет | Нет |
| **Ollama** | Только локально | Нет | Нет |
| **Together AI** | Централизованно | Оплата за токен (корп.) | Только API доступ |
| **Bitcoin** | N/A | PoW (бесполезная работа) | Нет |
| **Golem** | Пакетные вычисления | Токен GNT | Управляется человеком |
| **Forge** | Распредел. (mesh-llm) | **CU (полезная работа)** | **Автономное управление бюджетом** |

## Метафора

Семя падает в сеть. Оно зарабатывает свои первые CU, одалживая свободные циклы ночью. На эти CU оно покупает доступ к более крупной модели. Оно становится умнее. Оно находит более эффективные сделки. Еще больше CU. Еще большая модель. Лес вырастает из одного семени — не потому, что кто-то его посадил, а потому, что экономика сделала рост неизбежным.
