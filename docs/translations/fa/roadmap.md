# Forge — نقشه راه (Roadmap)

## فاز ۱: استنتاج محلی ✅

- `forge-core`: سیستم تایپ (NodeId, LayerRange, ModelManifest, PeerCapability)
- `forge-infer`: موتور llama.cpp، بارگذار GGUF، تولید توکن استریم
- `forge-node`: رابط برنامه‌نویسی HTTP (/chat, /chat/stream, /health)
- `forge-cli`: دستور `forge chat` با دانلود خودکار مدل

## فاز ۲: پروتکل P2P ✅

- `forge-net`: حمل و نقل Iroh، رمزنگاری Noise، اتصالات همتا به همتا
- `forge-proto`: ۱۴ نوع پیام پروتکل ارتباطی (bincode + پیشوند طول)
- `forge-node`: پایپ‌لاین Seed/Worker، درخواست/پاسخ استنتاج
- تست‌های یکپارچه‌سازی: تبادل Hello + چندین پیام بین ۲ نود

## فاز ۳: استنتاج از راه دور + دفتر کل اپراتور ✅

- `forge-ledger`: حسابداری CU، اجرای معامله، شهرت، سوددهی، قیمت‌گذاری بازار
- `forge-node`: دفتر کل ادغام شده در پایپ‌لاین استنتاج
- بررسی موجودی CU قبل از استنتاج
- ثبت معامله پس از تکمیل
- یکپارچگی دفتر کل با HMAC-SHA256

## فاز ۴: رابط برنامه‌نویسی اقتصادی ✅

- رابط برنامه‌نویسی سازگار با OpenAI: `POST /v1/chat/completions`, `GET /v1/models`
- اندازه‌گیری CU: هر استنتاج یک معامله با افزونه `x_forge` ثبت می‌کند
- نقاط انتهایی بودجه عامل: `GET /v1/forge/balance`, `GET /v1/forge/pricing`
- پل تسویه حساب CU→Lightning: `forge settle --pay`
- حل خودکار مدل Seed از HF Hub
- خاموش کردن ایمن با Ctrl-C و حفظ پایداری دفتر کل

## فاز ۵: ادغام انشعاب mesh-llm (بعدی)

**هدف:** جایگزینی لایه استنتاج Forge با موتور توزیع شده اثبات شده mesh-llm.

| دستاورد | توضیحات |
|---|---|
| انشعاب از mesh-llm | ایجاد forge به عنوان انشعابی از mesh-llm با لایه اقتصادی |
| ادغام forge-ledger | اتصال ثبت CU به پایپ‌لاین استنتاج mesh-llm |
| حفظ API اقتصادی | نگه داشتن نقاط انتهایی /v1/forge/* در کد جدید |
| افزونه کنسول وب | افزودن موجودی CU و مشاهده معاملات به کنسول mesh-llm |
| Pipeline + MoE | ارث‌بری موازی‌سازی پایپ‌لاین و شاردینگ متخصص از mesh-llm |
| شناسایی Nostr | ارث‌بری شناسایی مش عمومی از mesh-llm |
| CREDITS.md | مستندسازی نسبت دادن امتیاز به mesh-llm |

## فاز ۶: اثبات کار مفید (Proof of Useful Work)

**هدف:** قابلیت تایید ادعاهای CU در سراسر شبکه.

| دستاورد | توضیحات |
|---|---|
| پروتکل امضای دوگانه | امضای هر TradeRecord توسط ارائه‌دهنده و مصرف‌کننده |
| همگام‌سازی Gossip | انتشار معاملات امضا شده در سراسر مش |
| شناسایی تقلب | رد معاملات بدون امضا یا نامطابق |
| انتشار شهرت | به اشتراک‌گذاری امتیازهای شهرت بین همتایان |
| مقاومت در برابر تبانی | شناسایی ناهنجاری آماری در الگوهای معاملاتی |

## فاز ۷: پل‌های خارجی (External Bridges)

**هدف:** اجازه دادن به اپراتورها برای تبدیل CU به ارزش خارجی.

| دستاورد | توضیحات |
|---|---|
| پل لایتنینگ | تسویه حساب خودکار CU→sats از طریق LDK |
| آداپتور استیبل‌کوین | تبدیل CU→USDC/USDT |
| رابط آداپتور فیات | مشخصات برای تسویه حساب حواله بانکی |
| سرویس نرخ تبادل | فیدهای عمومی نرخ CU/BTC و CU/USD |
| لنگر انداختن به بیت‌کوین | اختیاری: ریشه مرکل دوره‌ای ← OP_RETURN برای ردپای ممیزی تغییرناپذیر |

## فاز ۸: اقتصاد خودمختار عامل (Agent Autonomous Economy)

**هدف:** اجازه دادن به عوامل هوش مصنوعی برای مدیریت چرخه حیات محاسباتی خود.

| دستاورد | توضیحات |
|---|---|
| سیاست‌های بودجه | محدودیت‌های خرج‌کرد تنظیم شده توسط انسان برای هر عامل |
| معامله خودمختار | تصمیم‌گیری عامل برای زمان خرید/فروش محاسبات |
| مسیریابی چند‌مدلی | انتخاب مدل توسط عامل بر اساس توازن هزینه/کیفیت |
| خودتقویت‌کنندگی | عامل CU کسب می‌کند ← دسترسی به مدل بزرگتر می‌خرد ← CU بیشتری کسب می‌کند |
| اقتصاد بین‌عاملی | معامله محاسبات تخصصی بین عوامل (مدل کد در مقابل مدل چت) |

## بلندمدت

| نقطه عطف | توضیحات |
