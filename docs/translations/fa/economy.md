# Forge — مدل اقتصادی (Economic Model)

## استاندارد محاسباتی (Compute Standard - 計算本位制)

هر سیستم پولی توسط کمیابی (scarcity) پشتیبانی می‌شود:

| عصر | استاندارد | پشتوانه |
|-----|----------|---------|
| باستان | طلا/نقره | کمیابی زمین‌شناسی |
| ۱۹۴۴–۱۹۷۱ | برتون وودز | دلار متصل به طلا |
| ۱۹۷۱–تاکنون | پترودلار | تقاضای نفت + قدرت نظامی |
| ۲۰۰۹–تاکنون | بیت‌کوین | الکتریسیته سوخته شده برای SHA-256 |
| **Forge** | **استاندارد محاسباتی** | **الکتریسیته صرف شده برای استنتاج مفید** |

Forge یک استاندارد محاسباتی معرفی می‌کند: واحد ارزش توسط هزینه واقعی انرژی برای انجام محاسبات مفید پشتیبانی می‌شود. برخلاف اثبات کار (Proof of Work) بیت‌کوین، هر ژول انرژی صرف شده در Forge هوش واقعی تولید می‌کند.

## CU: ارز بومی

### CU چیست

**۱ CU = ۱ میلیارد عملیات ممیز شناور (FLOPs) کار استنتاج تایید شده.**

CU یک ارز دیجیتال (cryptocurrency) نیست. یک توکن روی بلاک‌چین نیست. این یک واحد حسابداری است که نشان‌دهنده محاسبات واقعی انجام شده است. CU ارزش دارد زیرا ادعایی بر محاسبات آینده است — اگر با ارائه استنتاج CU کسب کردید، می‌توانید آن را برای دریافت استنتاج خرج کنید.

### چرا CU، نه بیت‌کوین

| ویژگی | CU | بیت‌کوین |
|----------|-----|---------|
| **پشتوانه ارزش** | محاسبات مفید (ذاتی) | محاسبات هش (مصنوعی) |
| **سرعت تسویه** | آنی (دفتر کل محلی) | ثانیه تا دقیقه (لایتنینگ/زنجیره) |
| **هزینه تراکنش** | صفر | هزینه‌های کانال، هزینه‌های زنجیره‌ای |
| **وابستگی خارجی** | ندارد | سلامت شبکه بیت‌کوین |
| **خطر کوانتومی** | ندارد (بدون معمای رمزنگاری) | SHA-256 / ECDSA آسیب‌پذیر |
| **تولید سود** | بله (سخت‌افزار بیکار CU کسب می‌کند) | خیر (بیت‌کوین در کیف پول سودی ندارد) |

CU واحد تسویه حساب **اصلی** است. بیت‌کوین، استیبل‌کوین‌ها و ارزهای فیات **مسیرهای خروجی (off-ramps)** اختیاری هستند که از طریق آداپتورهای پل خارج از پروتکل در دسترس می‌باشند.

### CU به عنوان یک دارایی مولد

```
ساختمان آپارتمانی            مک مینی در Forge
───────────────────         ──────────────────
دارایی: ساختمان             دارایی: سخت‌افزار محاسباتی
هزینه: نگهداری              هزینه: الکتریسیته
درآمد: اجاره                درآمد: CU از استنتاج
سود: اجاره - هزینه نگهداری   سود: CU کسب شده - هزینه الکتریسیته
بیکار ماندن = درآمد از دست رفته  بیکار ماندن = پتانسیل هدر رفته
```

یک دستگاه محاسباتی در Forge مانند بیت‌کوین در کیف پول نیست (ارزش ایستا، بدون سود). مانند یک ملک اجاره‌ای است — از طریق کار مفید درآمد ایجاد می‌کند.

## مدل تراکنش

### اجرای معامله

هر استنتاج یک معامله بین دو طرف ایجاد می‌کند:

```rust
pub struct TradeRecord {
    pub provider: NodeId,       // چه کسی استنتاج را اجرا کرد
    pub consumer: NodeId,       // چه کسی درخواست داد
    pub cu_amount: u64,         // مقدار CU منتقل شده
    pub tokens_processed: u64,  // کار انجام شده
    pub timestamp: u64,
    pub model_id: String,
}
```

معامله توسط هر دو طرف ثبت می‌شود. در پیاده‌سازی فعلی، هر نود یک دفتر کل محلی را نگهداری می‌کند. پیاده‌سازی هدف، امضاهای دوگانه و همگام‌سازی (gossip sync) را اضافه می‌کند.

### قیمت‌گذاری پویا

قیمت‌های CU بر اساس عرضه و تقاضای محلی نوسان می‌کنند:

```
effective_price = base_cu_per_token × demand_factor / supply_factor
```

- **نودهای بیکار بیشتر** ← ضریب عرضه افزایش می‌یابد ← قیمت کاهش می‌یابد
- **درخواست‌های استنتاج بیشتر** ← ضریب تقاضا افزایش می‌یابد ← قیمت افزایش می‌یابد
- هر نود شرایط بازار خود را مشاهده می‌کند. هیچ دفتر سفارش جهانی وجود ندارد.

### سطح رایگان (Free Tier)

نودهای جدید بدون سابقه مشارکت، ۱,۰۰۰ CU دریافت می‌کنند. این به هر کسی اجازه می‌دهد بلافاصله از شبکه استفاده کند. سطح رایگان از اولین درخواست مصرف می‌شود و دوباره تنظیم نمی‌شود.

مقابله با حمله سیبیل (Sybil): اگر بیش از ۱۰۰ نود ناشناخته بدون مشارکت ظاهر شده باشند، درخواست‌های جدید سطح رایگان رد می‌شوند.

## اثبات کار مفید (Proof of Useful Work)

### مفهوم

اثبات کار بیت‌کوین: "من الکتریسیته را برای محاسبه هش‌های SHA-256 سوزاندم. این نانس (nonce) است که آن را ثابت می‌کند."

اثبات کار مفید Forge: "من الکتریسیته را برای اجرای استنتاج LLM سوزاندم. این پاسخ است، و این امضای مصرف‌کننده است که دریافت آن را تایید می‌کند."

تفاوت کلیدی: اثبات بیت‌کوین خودساخته است (هر ماینری می‌تواند هش معتبر تولید کند). اثبات Forge به یک **طرف مقابل** نیاز دارد — کسی که واقعاً آن استنتاج را می‌خواسته است. شما نمی‌توانید تقاضا را جعل کنید.

### پروتکل تایید (هدف)

```
1. مصرف‌کننده InferenceRequest را به ارائه‌دهنده ارسال می‌کند
2. ارائه‌دهنده استنتاج را اجرا کرده و توکن‌ها را استریم می‌کند
3. مصرف‌کننده توکن‌ها را دریافت کرده و هش پاسخ را محاسبه می‌کند
4. هر دو طرف TradeRecord را امضا می‌کنند:
   - ارائه‌دهنده امضا می‌کند: "من این را محاسبه کردم"
   - مصرف‌کننده امضا می‌کند: "من این را دریافت کردم"
5. TradeRecord با امضای دوگانه در شبکه پخش (gossip-synced) می‌شود
6. هر نودی می‌تواند هر دو امضا را تایید کند
```

یک نود نمی‌تواند بدون همکاری طرف مقابل، موجودی CU خود را افزایش دهد. تبانی امکان‌پذیر است اما از نظر اقتصادی غیرمنطقی است — مصرف‌کننده تبانی‌کننده با امضای معاملات جعلی چیزی به دست نمی‌آورد.

### پیاده‌سازی فعلی

پیاده‌سازی مرجع فعلی از دفترهای کل محلی با محافظت یکپارچگی HMAC-SHA256 استفاده می‌کند. امضاهای دوگانه و پخش شبکه گام بعدی هستند.

## سوددهی و شهرت (Yield and Reputation)

### سوددهی (Yield)

نودهایی که آنلاین می‌مانند و محاسبات ارائه می‌دهند، سود کسب می‌کنند:

```
yield_cu = contributed_cu × 0.001 × reputation × uptime_hours
```

در شهرت ۱.۰، نودی با ۱۰,۰۰۰ CU مشارکت، ۸۰ CU در هر ۸ ساعت شب کسب می‌کند. این تورم نیست — پاداشی برای در دسترس بودن است. نودهایی که به‌طور قابل اعتمادی آنلاین هستند برای شبکه ارزشمندترند.

### شهرت (Reputation)

هر نود دارای امتیاز شهرت بین ۰.۰ تا ۱.۰ است:

- نودهای جدید از ۰.۵ شروع می‌کنند
- زمان آنلاین بودن (Uptime) و معاملات موفق شهرت را افزایش می‌دهند
- قطع اتصال‌ها و معاملات ناموفق شهرت را کاهش می‌دهند
- شهرت بالاتر ← نرخ سود بالاتر، اولویت در زمان‌بندی

## تسویه حساب و پل‌های خارجی

### قانون اصلی

**پروتکل با CU تسویه حساب می‌کند.** تبدیل به هر چیز دیگری یک موضوع یکپارچه‌سازی است.

### صورت‌حساب‌های تسویه

اپراتورها می‌توانند تاریخچه معاملات قابل بازرسی را برای هر بازه زمانی خروجی بگیرند:

```
forge settle --hours 24 --price 0.05 --out settlement.json
```

صورت‌حساب شامل این موارد است: مجموع CU کسب شده، مجموع CU خرج شده، خالص CU، تعداد معاملات و قیمت مرجع اختیاری به ازای هر CU.

### معماری پل (Bridge)

```
لایه ۰: پروتکل Forge
  → حسابداری CU، معاملات، قیمت‌گذاری

لایه ۱: صورت‌حساب تسویه
  → تاریخچه معاملات قابل خروجی
  → نرخ تبادل مرجع

لایه ۲: پل خارجی (اختیاری)
  → CU ↔ BTC (لایتنینگ)
  → CU ↔ استیبل‌کوین
  → CU ↔ فیات
```

لایه پل خارج از پروتکل است. اپراتورهای مختلف می‌توانند از پل‌های متفاوتی استفاده کنند. پروتکل حتی با نقدینگی خارجی صفر نیز مفید باقی می‌ماند.

### پل لایتنینگ (Lightning Bridge)

برای اپراتورهایی که خواهان تسویه حساب بیت‌کوین هستند:

```bash
forge settle --hours 24 --pay
```

این یک فاکتور لایتنینگ BOLT11 برای خالص CU کسب شده ایجاد می‌کند که با نرخ تبادل پیکربندی شده (پیش‌فرض: ۱۰ msats به ازای هر CU) تبدیل شده است.

## بودجه‌های تحت هدایت عامل (Agent-Directed Budgets)

### چشم‌انداز

سنتی: انسان تصمیم می‌گیرد ← انسان پرداخت می‌کند ← هوش مصنوعی اجرا می‌کند
Forge: سیاست به عامل اجازه می‌دهد ← عامل بودجه را بررسی می‌کند ← عامل CU خرج می‌کند ← عامل اجرا می‌کند

### API

```
GET /v1/forge/balance   → موجودی CU، مشارکت، مصرف، شهرت
GET /v1/forge/pricing   → قیمت بازار، تخمین هزینه به ازای هر ۱۰۰/۱۰۰۰ توکن
```

یک عامل می‌تواند:
۱. قبل از ارسال درخواست، موجودی خود را بررسی کند
۲. هزینه استنتاج را با قیمت‌های فعلی بازار تخمین بزند
۳. تصمیم بگیرد که آیا درخواست ارزش هزینه CU را دارد یا خیر
۴. به‌طور خودکار اجرا و پرداخت کند

ناظران انسانی سیاست‌های بودجه را تنظیم می‌کنند. عوامل در آن محدودیت‌ها به‌طور خودمختار عمل می‌کنند.

### چرخه خودتقویت‌کننده

```
عامل (کوچک، تلفن)
  ← با قرض دادن محاسبات بیکار CU کسب می‌کند
  ← برای دسترسی به مدل بزرگتر CU خرج می‌کند
  ← هوشمندتر می‌شود
  ← تصمیمات اقتصادی بهتری می‌گیرد
  ← CU بیشتری کسب می‌کند
  ← به مدل‌های حتی بزرگتر دسترسی پیدا می‌کند
  ← ...
```

این یک الگوی کاربردی ممکن است. پروتکل بازار را فراهم می‌کند؛ عوامل استراتژی را.

## چرا این وب۳ (Web3) نیست

بیشتر پروژه‌های وب۳ کمیابی مصنوعی (توکن) را روی کالاهای دیجیتال فراوان ایجاد می‌کنند. Forge عکس این کار را انجام می‌دهد:

- **محاسبات واقعاً کمیاب است** — به الکتریسیته واقعی، سیلیکون واقعی و زمان واقعی نیاز دارد
- **CU سفته‌بازی نیست** — نشان‌دهنده کار تایید شده است، نه شرط‌بندی روی پذیرش در آینده
- **بدون ICO، بدون فروش توکن، بدون توکن حاکمیتی** — CU با کار کردن به دست می‌آید
- **نیازی به بلاک‌چین نیست** — امضاهای دوطرفه و پخش شبکه کافی هستند
- **بدون قرارداد هوشمند** — پروتکل خودش قرارداد است

ارزش توسط فیزیک تولید می‌شود، نه با اجماع.
